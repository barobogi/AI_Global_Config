## 2. 개선안 3가지

### ✅ 개선안 1: API 사용량 실시간 모니터링 강화
**목표**: 사용량 70% 도달 시 자동 경고 → 80% 도달 시 에러율 높은 기능 자동 비활성화

**구현**:
```python
# Global_Define/anthropic_admin_api.py 추가
def check_api_quota():
    usage_pct = get_current_usage()
    
    if usage_pct >= 80:
        # 텔레그램 워치독 일시 중지
        disable_telegram_watchdog()
        send_alert("⚠️ API 사용량 80% 도달 — 텔레그램 차단")
    elif usage_pct >= 70:
        # 경고만 (계속 모니터링)
        send_alert(f"📊 API 사용량 {usage_pct}% 도달")
```

**산출물**:
- 수정: `Global_Define/anthropic_admin_api.py`
- 신규: `Global_Define/api_quota_dashboard.html` (실시간 사용량 표시)

### ✅ 개선안 2: 워치독 격리 및 독립적 재시작
**목표**: 텔레그램 워치독 문제가 원격제어 워치독을 영향 주지 않도록 격리

**구현**:
```python
# master_watch.py 리팩토링
class WatchdogManager:
    def __init__(self):
        self.telegram_watchdog = TelegramWatchdog()
        self.remote_control_watchdog = RemoteControlWatchdog()
        
    def start(self):
        # 각 워치독을 독립적 Thread에서 실행
        Thread(target=self._isolated_run, args=(self.telegram_watchdog,)).start()
        Thread(target=self._isolated_run, args=(self.remote_control_watchdog,)).start()
    
    def _isolated_run(self, watchdog):
        # 한쪽 실패가 다른 쪽에 영향 주지 않음
        try:
            watchdog.run()
        except Exception as e:
            logging.error(f"Watchdog {watchdog.name} failed: {e}")
            # 격리된 에러 처리
```

**산출물**:
- 수정: `Global_Define/master_watch.py` (멀티스레드 격리)
