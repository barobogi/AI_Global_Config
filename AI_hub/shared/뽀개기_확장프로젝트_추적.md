단순 "상태를 보고받고 믿는" 것을 넘어, 아래 4가지 기법을 실제 적용해 효과를 확인함:

1. **히든 로그 대조법** — 완료 보고에 안 나오는 백그라운드 로그(`Global_Define\global_watcher.log`)를 직접 확인. → **실제 발견**: `git add .`가 exit 128로 반복 실패 중이던 것 포착(만복에게 알림).
2. **코드 실체 대조법** — "구현했다"는 말을 믿지 않고 TODO/주석 처리된 줄을 직접 대조. → T022에서 실제 미연동 부분 발견(이후 실제 스캔은 별도로 진행되어 진행 확인됨).

4. **적대적/실패 시나리오 요구** — 성공 케이스 로그만 받지 말고 "반려 누르면", "토큰 만료되면" 같은 실패 시나리오 로그도 별도 요구 — T020 Timeout(24h) 구현 요구로 이미 반영 중.
