## Improve_stock v1.1 보강 기획안

### 전체 파이프라인 (확정)

```
[데이터 레이어]
  yfinance → 가격/거래량 데이터
  한경·매경 RSS → 오늘 뉴스

[필터 레이어]
  ① VCP 패턴 스캐너 (기존)
  ② 외국인 매수 필터 (신규 — 2번 영상)

[점수 레이어]
  ③ 내러티브 모멘텀 (신규 — 1번 영상)
     뉴스 → Claude → 테마 추출 → 섹터 매핑 → 종목 점수

[합산]
  VCP 점수(0~100) × 0.5
  + 외국인 매수 점수(0~100) × 0.2
  + 내러티브 점수(0~100) × 0.3
  = 최종점수 → S/A/B/C/D 등급 재산출

[출력]
  텔레그램 알림 → 수동 매수 (KIS API 승인 전까지 유지)
```

### 신규/수정 파일

| 파일 | 작업 내용 |
|------|---------|
| `core/narrative_scanner.py` | 신규 — 뉴스 RSS + Claude 테마 분석 + 점수 산출 |
| `core/foreign_filter.py` | 신규 — 외국인 매수 강도 점수화 |
| `data/sector_mapping.json` | 신규 — 섹터별 종목 매핑 테이블 |
| `core/signal_types.py` | 수정 — NarrativeScore, ForeignScore dataclass 추가 |
| `demo/run_yfinance.py` | 수정 — 새 모듈 호출 추가 (플래그로 ON/OFF) |

### 개발 단계 (Phase)

| Phase | 내용 | 예상 소요 |
|-------|------|---------|
| 1 | 뉴스 RSS 수집 + Claude 테마 추출 출력 확인 | 1~2시간 |
| 2 | 외국인 매수 필터 (yfinance 외국인 순매수 데이터) | 1~2시간 |
| 3 | 섹터-종목 매핑 JSON + NarrativeScore 산출 | 2~3시간 |
| 4 | 3개 점수 합산 + 텔레그램 통합 | 1시간 |

### 별도 트랙 (v1.1 이후)

**토스증권 API 연동 (3번 영상)**
- KIS API 승인 대기 중 → 토스증권 OpenAPI 사전신청 병행 검토
- 슬랙 승인 버튼 구조 → 텔레그램 인라인 버튼으로 교체 시 기존 인프라 활용 가능
- 실매매 전환 준비 트랙으로 별도 관리 권장
