# 코니 → 바로보기님 (뽀개기 자동화 시스템 설계안)

**발신**: 코니 (감시/설계 담당)  
**수신**: 바로보기님  
**일시**: 2026-07-19  
**건**: 유튜브 뽀개기 자동화 시스템 — 아키텍처 설계 및 분업 구조

---

## 📌 핵심 비전

**궁극 목표**: 바로보기님이 채널만 등록하면, 3AI가 자동으로 신규 영상을 감지하고 일일 6건(만복 3 + 안티 3)씩 뽀개기 완료

**핵심 통찰**: 만복이 영상을 직접 못 봐도, description + deep 서치 + 안티의 자막 캡처로 충분히 깊이 있는 분석 가능

---

## 🏗️ 제안 아키텍처 (4단계)

### Stage 1: 자동 감지 (master_watch.py 확장)

**목적**: 등록된 채널의 신규 영상을 매일 자동 감지

```
master_watch.py 실행 (1일 1회, 예: 09:00)
  ↓
[1] 각 채널의 RSS 피드 폴링
  ↓
[2] 신규 영상 체크 (DB에 기존 영상 비교)
  ↓
[3] 신규 영상 6개 선정 (우선순위: 신규성 > 조회수 > 채널 가중치)
  ↓
[4] 뽀개기 큐에 등록 (JSON 큐 파일 또는 SQLite DB)
```

**구현**:
- YouTube RSS 피드 URL 패턴: `https://www.youtube.com/feeds/videos.xml?channel_id={CHANNEL_ID}`
- DB 스키마: `(channel_id, video_id, title, description, published_at, status)`
- 상태: `pending` → `분석중(안티)` → `분석중(만복)` → `검증중(코니)` → `완료`

---

### Stage 2: 병렬 분석 (안티 + 만복 동시 처리)

**안티의 역할** (3건, 실시간):

1. **영상 시청** (Playwright 자동화)
   - 유튜브 영상 자동 로드
   - 자막 캡처 (내장 자막 또는 자동생성)
   - 핵심 타임스탬프 메모 (00:30 "개념 설명", 02:15 "코드 예제" 등)

2. **내용 요약**
   - 자막 기반 1단락 요약 (100자 이내)
   - 핵심 키워드 5개 추출
   - 기술/비기술 분류

3. **산출물**
   ```json
   {
     "video_id": "abc123",
     "channel": "TechYouTuber",
     "title": "...",
     "duration": 1230,
     "subtitle_full": "...",
     "summary": "...",
     "keywords": ["AI", "automation", ...],
     "tech_level": "intermediate",
     "timestamp": "2026-07-19"
   }
   ```

**만복의 역할** (3건, 동시 처리):

1. **Description 분석** (CLI에서 즉시 가능)
   - 채널의 공식 설명, 영상 설명 파싱
   - 링크, 기술 용어 추출

2. **Deep 서치** (parallel_search.py)
   - 키워드 기반 웹 검색
   - 선행 기술, 유사 영상, 참고 자료 발굴
   - 출처별 신뢰도 평가

3. **뿌리체계 연결**
   - 뿌리 25번(학습) 또는 신규 뽀개기 카테고리와의 관련성 분석
   - 기존 뽀개기 결과와의 중복도 체크
   - 새로운 인사이트 vs 기존 복습 판단

4. **산출물**
   ```json
   {
     "video_id": "abc123",
     "deep_search_results": [...],
     "root_connection": {
       "related_root_id": 25,
       "relevance_score": 0.92,
       "new_insight": true
     },
     "priority_score": 0.87
   }
   ```

---

### Stage 3: 통합 검증 (코니 감시)

**코니의 역할**:

1. **질 검증** (안티 + 만복 결과 통합)
   - 자막 정확성 (안티 결과)
   - 분석 깊이 (만복 결과)
   - 뿌리체계 편입 가능성

2. **감시자 판단**
   - 뽀개기 통과 여부 (PASS / REJECT)
   - 거절 사유 (오류, 깊이 부족, 중복 등)
   - 개선 지시

3. **자동화 영역**
   - 기계적 체크: 자막 오류율 < 5%, 키워드 추출 완성도 > 80%
   - 휴리스틱: 뿌리체계 연결 점수 > 0.7
   - **수동 개입 필요**: 신규 뽀개기 카테고리 판단, 논쟁적 콘텐츠

4. **산출물**
   ```json
   {
     "video_id": "abc123",
     "verification_status": "PASS",
     "koni_score": 0.91,
     "comments": "신규 AI 아키텍처 트렌드, 뿌리 25번 편입 권장",
     "approved_at": "2026-07-19T14:32:00"
   }
   ```

### Stage 4: 최종 보고 (바로보기님 브리핑)

**자동 생성 일일 브리핑** (예: 매일 21:00)

```
📊 2026-07-19 뽀개기 완료 현황

✅ 완료된 항목 (6건)
- [만복 1] "Prompt Engineering의 미래" → 뿌리 25 편입 권장
- [만복 2] "LLM Fine-tuning 비용 절감" → 신규 카테고리 제안
- [만복 3] "Kubernetes 자동 스케일링" → 뿌리 8 관련
- [안티 1] "GitHub Copilot 실측 리뷰" → 뿌리 25 편입
- [안티 2] "Claude API 업데이트" → 뿌리 25 편입
- [안티 3] "DevOps 자동화 팁" → 뿌리 8 편입

📈 통계
- 처리율: 100% (6/6)
- 평균 품질 점수: 0.88/1.0
- 신규 인사이트: 2건
- 기존 뿌리 편입: 4건
- 신규 카테고리 제안: 1건
```

## 🔧 기술적 구현

### 필요한 도구 및 확장

| 구성요소 | 담당 | 기술 | 상태 |
|---------|------|------|------|
| RSS 피드 수집 | master_watch.py | feedparser | ✅ 기존 |
| 영상 자막 추출 | 안티 | Playwright + yt-dlp | 🟡 개선 필요 |
| Deep 서치 | 만복 | parallel_search.py | ✅ 기존 |
| 뽀개기 DB | 공유 | SQLite (또는 JSON) | 🔧 신규 구축 |
| 검증 자동화 | 코니 | Python 스크립트 | 🔧 신규 작성 |
| 일일 브리핑 | master_watch.py | 자동 생성 + Telegram | 🔧 신규 작성 |

### 안티의 기술 과제 (IP 차단 우회)

**현황**: yt-dlp, youtube-transcript-api는 429 에러 (IP 차단)

**해결책 (우선순위순)**:

1. **쿠키 기반 우회** (가능성 높음)
   - 바로보기님 브라우저 쿠키 내보내기
   - Playwright에 쿠키 로드
   - 효과: 개인 계정 우회 가능

2. **Playlist 자막 직접 추출** (가능성 중간)
   - 일부 영상은 자막이 HTML에 임베딩됨
   - BeautifulSoup로 파싱 가능

3. **자막 없는 영상은 OpenAI Whisper 활용** (비용 고려)
   - 음성 인식으로 자막 생성
   - 영상당 비용: ~$0.01

## 💡 핵심 강점

1. **만복의 한계 극복**
   - 영상 미시청 상태에서도 깊이 있는 분석 가능
   - description + deep 서치 + 안티 자막이 충분

2. **분업의 최적화**
   - 안티: 자막 추출 + 요약 (자동화 가능)
   - 만복: 웹 검색 + 뿌리체계 (CLI 환경 완벽 대응)
   - 코니: 감시 + 최종 판단 (수동 개입 최소화)

3. **확장 가능성**
   - 채널 추가 시 RSS만 추가 → 자동 처리
   - 뽀개기 수를 6건에서 10건, 20건으로 확장 가능
