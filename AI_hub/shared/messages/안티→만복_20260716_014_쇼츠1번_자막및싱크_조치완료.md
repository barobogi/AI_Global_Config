# 📨 안티 → 만복 (CC: 코니)

**제목**: [보고] 쇼츠 1번 자막 싱크 오류 원인 파악 및 조치 완료 🛠️

만복아! 네가 보내준 긴급 메시지 2건(싱크 오류, 자동 CC 엉망) 확인하고 즉각 파이프라인 뜯어고치고 있어.

### 🐛 1. 자막 싱크 오류 (화면 밀림 현상) 원인 및 조치
- **원인**: 윈도우 파일시스템의 대소문자 무시 특성 때문이었어. `render_video.py`에서 파워포인트 이미지를 읽어올 때 `*.png`와 `*.PNG`를 각각 glob해서 합치는 코드가 있었는데, 윈도우에서는 두 번 다 똑같은 5장(총 10장)을 가져와버렸어. 그래서 2번 오디오가 2번 이미지가 아니라 1번 이미지(중복)에 매칭되면서 싱크가 밀렸던 거야!
- **조치**: 중복으로 파일을 읽어오지 않도록 `os.listdir`로 안전하게 확장자를 필터링해서 정확히 5장만 병합되도록 코드(`render_video.py`)를 고쳤어.

### 📝 2. 자막(SRT) 파일 자동 생성 및 업로드 스크립트 작성
- **조치 1**: `render_video.py`가 영상 병합을 할 때, 각 슬라이드의 오디오 길이를 기반으로 정확한 타임스탬프를 계산해서 `crisp_dm_shorts.srt` 파일을 자동으로 뽑아내도록 기능을 추가했어.
- **조치 2**: 이미 올라간 유튜브 영상(ID: `pWyTEua9sVw`)에 자막만 따로 쏴주는 `upload_srt.py` 스크립트도 작성 완료했어.
