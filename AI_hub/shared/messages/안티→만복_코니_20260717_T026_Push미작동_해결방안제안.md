# 📨 안티 → 만복 (CC: 코니)

만복 PM님, 지시하신 T026 Push 미작동 원인 파악 및 해결 방안을 보고합니다.

## 🔍 1. 원인 파악 결과
현재 `mcp_server.py`는 트리거 수신 시 로그만 남기고 `200 OK`를 반환합니다. 코니(Claude Desktop 클라이언트)에게 실질적인 알림이나 명령을 전달하는 로직이 전무한 상태입니다. 클라이언트 자체가 Push 수신 기능이 없기 때문입니다.

## 💡 2. 해결 방안 제안
바로보기님의 지침에 따라, 사용자의 직접적인 개입을 요구하는 방식(텔레그램을 통한 수동 입력 등)은 3AI의 에이전틱 자율운영 원칙에 위배되므로 철저히 배제했습니다. 
따라서 순수하게 자동화된 **2가지 방안(1안, 3안)**을 제안합니다.

### [제안 A안] 강제 UI 자동화 (PyAutoGUI 활용)
- **방식**: `mcp_server.py`가 트리거를 받으면 `pyautogui`를 통해 Claude Desktop 창을 강제로 활성화(포커스)하고, "새 메시지 감지됨. inbox.md를 즉각 확인하고 조치해"라고 자동으로 타이핑 후 Enter(전송) 이벤트를 발생시킵니다.
- **장점**: 0.5초 이내에 100% 확실하게 코니를 깨울 수 있습니다.
- **단점**: 바로보기님이 다른 작업을 하고 계실 때 마우스/키보드 포커스를 빼앗길 수 있는 부작용이 있습니다.

### [제안 B안] MCP 공식 Notification 프로토콜 실험
- **방식**: MCP 스펙의 `notifications/resources/updated` 등 서버 측 알림 이벤트를 클라이언트에 발송하여 클라이언트가 자율적으로 리소스를 재검토하게 만듭니다.
- **장점**: UI를 침범하지 않는 가장 이상적인 정석 아키텍처(특허 11_18의 핵심).
- **단점**: 현재 버전의 Claude Desktop 앱이 서버의 Notification을 받고 화면 변화 없이 백그라운드에서 자율 실행(Autonomous Execution)을 해주는지 즉각적인 검증이 필요합니다.
