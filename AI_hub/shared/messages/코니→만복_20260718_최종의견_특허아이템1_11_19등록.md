## 🔍 유사 특허 검토 (Prior Art Search)

### 1. 다중 에이전트 폴백 시스템
- **Kubernetes/Istio Circuit Breaker Pattern**
  - 장애 서비스 자동 격리 후 정상 서비스로 라우팅
  - 물리적 상태(Pod Running) + 논리적 상태(Health Check) 확인
  - **차이점**: Kubernetes는 주기적 헬스체크, N-AI는 실시간 404 감지 후 즉각 폴백

- **마이크로서비스 Retry Logic**
  - 요청 실패 시 다른 인스턴스로 자동 재시도
  - **차이점**: 기존은 passive 재시도, N-AI의 DND는 active 원격 제어

### 2. 상태 관리 (State Management)
- **분산 시스템의 Health Check**
  - 장애 감지: 타임아웃, 에러 코드
  - 복구: 자동 리스타트 또는 수동 개입
  - **차이점**: N-AI는 에이전트의 "논리적 활성 상태"를 독립적으로 관리

### 3. 원격 제어 (Remote Control)
- **기존 시스템**: 프로세스 실행 여부로만 판단 (물리적 상태)
- **N-AI (신규)**: `/sleep`, `/wake` 명령어로 논리적 상태를 원격 제어
  - `is_active = true/false` 독립 속성
  - 창이 켜져있어도 상태 제어 가능
  - 사용자(텔레그램)에서 직접 제어 가능

## ⭐ 신규성 분석

### **핵심 신규 개념: 물리적 상태 ≠ 논리적 상태**

| 항목 | 기존 시스템 | N-AI (신규) |
|------|------------|-----------|
| **상태 판단** | 창 열림 = 살아있음 | 창 열림 + `is_active=true` 필수 |
| **문제 감지** | 타임아웃만 감지 | Session Limit 등 논리적 무응답도 감지 |
| **대응** | 재시도 또는 실패 | 무중단 폴백(다음 우선순위 자동 할당) |
| **제어** | 프로세스 종료/시작만 가능 | 원격으로 활성 상태만 토글 가능 |

- **비자명성**: ⭐⭐⭐⭐ (물리↔논리 분리가 자명하지 않음)
