## 2. 주요 구현 및 개선 내역

### ① 텔레그램 메시지 자동 편집 및 버튼 제거 기능 구현 (`send_approval.py`)
- `--edit-timeout` 옵션을 새롭게 추가하여 타임아웃 발생 시 호출되도록 구현하였습니다.
- 타임아웃 모드 동작 시, 텔레그램 API를 호출하여 기존 메시지의 인라인 버튼(승인/반려)을 즉시 제거하고, 메시지 본문을 `🔴 [승인 대기 시간 초과] 이 요청은 24시간 내에 응답이 없어 자동 반려되었습니다.` 상태로 깔끔하게 자동 수정합니다.
- `stdout` 반환 형식을 문자열 대신 JSON 형태(`{"status": "success", "chat_id": ..., "message_id": ...}`)로 통일하여 n8n의 후속 노드가 파싱하기 용이하도록 개선했습니다.

### ② n8n 워크플로우 타임아웃 분기 최적화 (`T020_approval_template.json`)
- **타임아웃 파라미터 적용**: Wait Node의 타임아웃 한계점 설정 변수를 n8n 내장 규격인 `resumeAmount`와 `resumeUnit`으로 명확히 연동하여, n8n 스케줄러가 정확한 시간에 깨어날 수 있도록 수정했습니다.
- **비대칭 흐름 격리**: 
  - 사용자가 1분(테스트용) 또는 24시간(실환경) 내에 직접 반려를 누르는 **수동 반려(Is Timeout?: True)** 흐름은 `Respond to Webhook` 노드로 이동하여 즉각 `HTTP 200` 및 안내 메시지를 콜백에 응답합니다.
  - 시간 초과로 강제 재개되는 **자동 반려(Is Timeout?: False)** 흐름은 웹훅 호출 주체가 없으므로 `Respond to Webhook` 노드를 거치지 않고, `Execute Timeout Action` 노드를 타서 독립적으로 텔레그램 메시지를 업데이트하게 격리하였습니다.

## 3. E2E 테스트 검증 결과

테스트의 신속성을 위해 임시로 **1분(minutes)** 타임아웃 설정을 주입한 뒤, n8n 서버와 봇 데몬을 재기동하여 아래와 같이 검증을 완수했습니다.

### ① 자동 반려(타임아웃) 검증 (`Execution ID: 7214`)
- Webhook trigger를 통해 유튜브 대본 승인 요청을 발송한 후 **1분간 무반응** 상태로 대기하였습니다.
- 정확히 1분 2초 후, n8n 스케줄러에 의해 대기 상태가 해제되었으며, `Is Approved?` (False) -> `Is Timeout?` (False) 분기를 지나 `Execute Timeout Action` 노드가 정상 작동했습니다.
- 텔레그램 메시지의 인라인 승인/반려 버튼이 완벽히 지워지고 본문이 `🔴 [승인 대기 시간 초과]`로 자동 편집된 것을 확인했습니다.
- SQLite DB 상에서 해당 실행건이 에러 없이 최종 `success` 상태로 완결 종료되었습니다.

### ② 수동 반려(1분 내 반려 클릭) 검증 (`Execution ID: 7219`)
- 신규 승인 요청 발송 후, 1분이 지나기 전에 텔레그램 메시지의 `[반려]` 버튼을 클릭했습니다.
- 봇 데몬이 반려 신호를 즉시 n8n Wait Node로 콜백하여 `HTTP 200` 정상 응답을 반환했습니다.
- n8n 워크플로우가 즉시 깨어나 `Is Approved?` (False) -> `Is Timeout?` (True) -> `Rejected Action` 경로를 거쳤으며, 텔레그램 메시지가 `🔴 [처리 완료] 반려`로 정상 업데이트되고 최종 `success`로 완결 종료되었습니다.
- 1분이 지난 뒤에도 중복 타임아웃 오작동이 발생하지 않음을 교차 확인하여 격리 완성도를 보장했습니다.
