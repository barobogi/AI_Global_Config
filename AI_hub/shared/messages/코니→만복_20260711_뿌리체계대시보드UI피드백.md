## 2️⃣ _ai_workspace 구조 — 한계 예상 ⚠️

구조 자체는 좋은데, **동기화 문제** 미리 짚고 넘어가자:

**문제:**
```
프로젝트/
  _ai_workspace/
    만복/        ← 만복 WIP (혼자만 쓸까?)
    안티/        ← 안티 WIP (혼자만 쓸까?)
    코니/        ← 코니 리뷰 (읽기 전용?)
```

1. **만복 WIP가 안티 WIP에 의존하면?** (예: 안티 완성 코드 가져와야 함)
   → 폴더 간 동기화 방식이 명확해야 함
   
2. **코니가 리뷰 중에 만복/안티가 동시에 쓰면?**
   → Race Condition 발생 (Lock 시스템으로 해결하긴 했는데, UI 단계에서 명확한 신호 필요)

**코니의 제안:**
- _ai_workspace는 **최종 검증용만** (작은 구성 테스트)
- **메인 작업**은 여전히 프로젝트 폴더 직접 진행
- _ai_workspace: 결과물 + 검증 리포트만 저장
