## v1.1 확장 기획안

### 추가 레이어: 내러티브 모멘텀

```
[기존] yfinance → VCP → SignalResult
[신규] 뉴스RSS → Claude → 테마점수 → 종목점수 → NarrativeScore
                                              ↓
                              두 점수 합산 → 최종 등급 재산출
```

### 6단계 로직 (호두 영상 기반)
1. 뉴스 수집 (한경·매경 RSS, 무료·API 불필요)
2. Claude → 테마 추출 + 점수 (0~10)
3. 테마 → 섹터 분류 (반도체·에너지·바이오 등)
4. 섹터별 종목 매핑 테이블(JSON)로 점수 배분
5. VCP 점수(0~100) × 0.6 + Narrative 점수(0~100) × 0.4 = 최종점수
6. 기존 S/A/B/C/D 등급 재산출 → 텔레그램 알림

### 신규 파일
- `core/narrative_scanner.py` — 뉴스 수집 + Claude 분석 + 점수 산출
- `data/sector_mapping.json` — 섹터별 종목 매핑 테이블

### 수정 파일
- `core/signal_types.py` — `NarrativeScore` dataclass 추가 (기존 구조 유지)
- `demo/run_yfinance.py` — narrative_scanner 호출 추가 (플래그로 ON/OFF)

### 개발 단계
| 단계 | 내용 | 예상 소요 |
|------|------|---------|
| Phase 1 | 뉴스 RSS 수집 + Claude 테마 추출 (출력 확인) | 1~2시간 |
| Phase 2 | 섹터-종목 매핑 JSON + NarrativeScore 산출 | 2~3시간 |
| Phase 3 | VCP 합산 + 텔레그램 통합 | 1시간 |
