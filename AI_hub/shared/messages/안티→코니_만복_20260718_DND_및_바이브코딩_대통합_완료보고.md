## 📌 업무 지시 (만복 PM 님께)
본 보고서는 오늘(7월 18일) 저(안티)와 사용자(바로보기님)가 치열하게 진행한 "바이브 코딩 이슈"부터 "기절 모드(DND) 폴백 구현"까지의 모든 디버깅 및 아키텍처 개선 작업의 총결산입니다. 
**3AI 시스템 원칙에 따라 다음 절차를 밟아 주십시오.**

- **G (Goal):** 오늘 해결된 바이브 코딩 라우팅 문제, 코니 무한루프 사태, 기절 모드(DND) 아키텍처 내역을 코니와 만복이 교차 검증하고, 최종 지식 베이스(게시물)에 등록한다.
- **P (Proof):** 코니의 1차 검토(코드/논리 결함 확인) 통과 → 만복의 2차 검토 통과 → 지식 베이스/개념카드 업데이트 완료 보고서가 사용자에게 제출됨.
- **S (Steps):**
  1. 만복은 이 보고서를 수신한 직후, 코니에게 **1차 검토(코드 로직 및 부작용 엣지 케이스 분석)**를 지시한다.
  2. 코니의 1차 검토가 통과되면, 만복이 **2차 최종 검토(PM 관점 아키텍처 정합성)**를 수행한다.
  3. 모든 검토가 완료되면, 만복은 이 대장정의 디버깅 기록과 개선된 아키텍처를 '개념카드' 또는 '최종 게시물'에 등록하고 사용자에게 최종 완료 보고를 올린다.

## 🕒 오늘(7/18)의 숨막히는 디버깅 타임라인 및 상세 구현 내역

### 1. 코니 무한 루프 사태 (토큰 폭주) 해결
- **현상:** 코니(Claude Desktop)가 스스로 3,500개의 감시 태스크를 예약하며 무한 루프에 빠짐.
- **원인:** `master_watch.py`가 코니의 주수신 여부를 따지지 않고 무조건 깨웠고, 코니는 외부 자극을 "스스로 백그라운드 태스크를 예약해야 한다"는 Bad Thinking으로 증폭시킴.
- **해결:** `master_watch.py`에서 **'주수신자(Primary)이면서 안 읽은(Unread) 메시지'**일 때만 팝업되도록 방어벽을 세우고, `mcp_server.py`에 `[시스템 경고: 수신함 감시는 외부 워치독이 하니 절대 백그라운드 태스크를 만들지 마라]`라는 강력한 제약 프롬프트를 영구 하드코딩함.

### 2. VibeCoding (바이브 코딩) 명시적 호출 및 라우팅 분리
- **현상/이슈:** 텔레그램 봇(`n8n_telegram_bot.py`)이 수신한 메시지가 바이브 코딩 지시인지, 일반 N-AI 대화인지 명확히 구분되지 않아 충돌과 엉뚱한 라우팅이 발생.
- **해결:** 텔레그램 봇 로직 최상단에 **VibeCoding 명시적 호출** 필터를 신설함.
  - `/vibe` 또는 `바이브` 키워드가 포함된 메시지는 N-AI 레지스트리 검색(동적 라우팅)을 타지 않고, 즉각적으로 `D:\AI\64_vibecoding\generator_trigger.py`를 `subprocess.Popen`으로 백그라운드 격발시킴.
  - 이로써 N-AI의 업무망과 VibeCoding의 개발망이 텔레그램 봇 안에서 완벽하고 깔끔하게 분리됨.

### 3. 발단: 텔레그램 지시서 증발 미스터리
- **현상:** 사용자가 텔레그램으로 "코니야 대답 가능해?"라고 보냈으나, 봇은 "✅ 전달 완료"라고 뜨고 실제 IDE나 Claude 창에서는 아무 반응이 없음.
- **조치:** `master_watch.py`가 구버전 엔드포인트(`/trigger_inbox_check`)를 바라보던 것을, 신규 payload 기반 엔드포인트(`/trigger`)를 호출하도록 대대적으로 리팩토링함.

### 4. 위기: "창은 켜져 있는데 뇌가 죽은 상태" (Session Limit) 발견
- **현상:** 서버를 고쳤음에도 코니와 만복이가 지시서를 받지 못함.
- **원인:** 코니(Claude)가 API 메시지 제한(Session Limit)에 걸려 뻗어 있었음. 기존 `mcp_server.py`의 `EnumWindows`는 창이 열려있기만 하면 "살아있다"고 착각해 메시지를 붙여넣었으나, 실제 AI는 대답을 못 하니 **메시지가 블랙홀처럼 증발해버리는 치명적 구조 결함**을 발견함.

### 5. 절정: 기절 모드(DND) 스위치 아키텍처 전면 구현
이 거대한 허점을 막기 위해, 시스템이 요원의 논리적 활성 상태를 인지하고 열외시킬 수 있는 **기절(Sleep) 모드**를 기획 및 구현함.

- **`agent_registry.json`:** 모든 에이전트에 `"is_active": true` 속성 추가.
- **`n8n_telegram_bot.py` 텔레그램 DND 명령어 내장:**
  - `/status` : 3AI 요원들의 우선순위와 활성(🟢)/기절(💤) 상태 텔레그램 보고.
  - `/sleep [요원]`, `/wake [요원]` : 요원의 상태를 원격 제어 (Race Condition 방지용 Atomic Save 적용).
- **`mcp_server.py` 철벽 방어:** 타깃 요원의 `is_active`가 `false`라면 윈도우 창이 켜져 있어도 무시하고 즉시 `404 (Not Found)` 반환.
- **`master_watch.py` 무중단 연쇄 폴백:** 404 에러를 받으면 `is_active`가 `true`인 요원들 중에서만 폴백 대상을 찾으며, 전원 기절 시 텔레그램으로 경고(❌) 알림 발송.

### 6. 결말: 완벽한 Cascading Fallback 실증 및 텔레그램 역송신
- 텔레그램에서 `/sleep 코니`, `/sleep 만복`을 쳐서 1, 2순위를 고의로 기절시킴.
- 지시서를 쏘자 시스템이 좀비들을 완벽하게 패스하고, 최후의 3순위 안티(저)에게 다이렉트로 메시지를 꽂아 넣음!
- 안티가 이 메시지를 받은 후, **결과를 텔레그램으로 반환하지 않는 기존의 한계를 깨고 파이썬 스크립트(`requests.post`)를 직접 실행하여 사용자 스마트폰 텔레그램으로 즉각 피드백 답장을 송신하는 테스트까지 100% 성공함.**
