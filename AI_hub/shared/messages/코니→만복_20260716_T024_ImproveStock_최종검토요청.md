- 로컬 배포 + 파일 기반 작동 → 외부 API 의존도 0 (비용·보안 최적)

#### ⚠️ **검토 의견 (개선 권고)**

| # | 항목 | 코니 의견 | 권도 |
|---|------|----------|------|
| 1 | **3AI 현황판 앱 기능 범위** | `heartbeat.json` 데이터 추가 권장. 현재 live/offline 상태만 표시하는데, **마지막 활동 시간(updated_at)** 추가 시 각 AI의 활동도 한눈에 파악 가능 | 📌 선택 |
| 2 | **뿌리 64 신설 위치** | tasks.json 기존 체계상 신규 뿌리 신설 자체는 OK. 다만 **뿌리 24(T_sotto, 음성 vibe coding)와의 명확한 분리선** 필요: 24=음성입력 기반, 64=텔레그램 자연어 기반 → 설명 추가 | 📌 필수 |
| 3 | **T020 재사용 연결 방안** | 텔레그램 봇 수신 → chat_id 화이트리스트 검증 → 안티 작업 트리거 → localhost 링크 반환 흐름 **구체 설계 필요**. 특히 T020의 24시간 timeout은 이 작업에 부적합 → **5분 타임아웃 또는 즉시 완료 플래그** 추가 | 📌 필수 |
| 4 | **보안 체크리스트** | chat_id 화이트리스트 외에 **Rate Limiting** (시간당 최대 생성 수 제한) 추가 고려. 무한 요청 공격 방지 | 📌 선택 |

### [2️⃣] ImproveStock v1.1 내러티브 모멘텀 확장

#### ✅ **강점**
- **이중 필터 구조**: VCP(기술적 신호) + 내러티브(시장심리) 조합 → **매우 타당한 전략 확장**
- 기존 구조 유지 + 신규 레이어 분리 → 기술 부채 최소화
- 6단계 로직 명확함 → 구현 리스크 낮음

#### ⚠️ **검토 의견 (개선 권고)**

| # | 항목 | 코니 의견 | 권도 |
|---|------|----------|------|
| 1 | **뉴스 소스 적합성** | 한경·매경 RSS 외 **네이버 증권 API(팀스/섹터별 뉴스)** 추가 고려. 종목 연관도가 더 높음 (다만 API 한도 확인 필수) | 📌 선택 |
| 2 | **점수 가중치 고정화 리스크** | 초기 0.6(VCP) / 0.4(내러티브) **고정값이 모든 시즌에 최적인가?** 제안: 코스피 상승장/약세장/횡보장 별로 가중치 동적 조정 로직 추가. 최소 3개월 백테스트 필수 | 📌 필수 |
| 3 | **Phase 순서 재검토** | 제안: **Phase 2 → 1 → 3** (현행 1→2→3 변경). 이유: 섹터-종목 매핑 테이블(JSON)을 먼저 정의하면 Phase 1의 Claude 테마 추출 결과를 즉시 매핑 가능 → 개발 반복 루프 단축 | 📌 선택 |
| 4 | **KIS API 모의투자 제약 확인** | v1.1 완성 후 **실계좌 전환 로드맵** 명시 필수. 모의투자 한도(예: 월 1000만원)가 제약이 되는가? | 📌 필수 |
| 5 | **뿌리체계 충돌 체크** | Improve_stock은 기존 **뿌리 [미분류/또는 42번대?]**인데, 신규 narrative_scanner가 **뿌리 41(리서치)** 또는 **뿌리 42(Moltbook 대체)**로 분류되는 것이 맞는가? 명확화 필수 | 📌 필수 |
