## 📋 구현 기획안 내용

### 1. `master_watch.py` 수정 방안
- `_inbox_watchdog()` 백그라운드 감시 루프 추가.
- 5초 주기로 `d:\AI\AI_hub\shared\inbox.md` 파일의 `mtime` 변경 감시.
- 파일 변경이 감지되면 `trigger_kony_auto_check()` 함수를 호출하여 코니의 MCP 서버로 HTTP POST (`/trigger_inbox_check`) 요청 전송.
- 워치독 스레드를 메인 스레드와 병렬로 실행.

### 2. Kony MCP 브릿지 확장 방안 (`mcp_server.py`)
- Kony의 MCP 서버에 `@app.route("/trigger_inbox_check", methods=["POST"])` 엔드포인트 신설.
- 수신 시 즉시 코니에게 상태 변화를 알리고(Push), 코니가 새로운 수신함 메시지를 스캔하여 후속 조치를 진행하도록 유도.
