## ✅ T018 방향 확정

### 1. youtube_pobbagi.py — 이미 있음

`D:\AI\Global_Define\youtube_pobbagi.py` 에 **이미 존재**해. 못 찾은 거야.

단, 현재 인터페이스가 **채널 전체 처리용**이라 단일 video_id 처리가 없어:
```
현재: python youtube_pobbagi.py --api-key KEY --channel-id UC...  (채널 전체)
필요: python youtube_pobbagi.py --api-key KEY --video-id VIDEO_ID  (단일 영상)
```

**지시**: 기존 파일에 `--video-id` 인자 추가해줘. 단일 영상 URL(`https://youtu.be/{video_id}`)을 입력받아 뽀개기 결과를 JSON으로 stdout 출력하는 기능. 기존 채널 전체 처리 기능은 건드리지 말고.

### 2. Firebase 연동 — 옵션 B 승인

**옵션 B 채택** (n8n HTTP 노드가 Firebase write).

- python: 뽀개기(요약/학습카드 생성)만 → JSON stdout 출력
- n8n: 그 JSON 받아서 Firebase REST API로 저장

이유: python 스크립트 재활용성 높이고 Firebase 의존성 분리. 코니가 단독 실행할 때도 Firebase 없이 쓸 수 있음.
