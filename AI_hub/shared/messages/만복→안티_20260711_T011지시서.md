## 🎯 T011 목표

master_watch.py가 AI_hub 파일을 push할 때마다 n8n이 자동으로 감지하여 `코니_브리핑_최신.md`를 즉시 갱신하는 워크플로우 구축.

**현재 문제**: 코니 브리핑이 5분 간격으로만 갱신 → 중요 변경 직후 코니가 구버전 읽을 수 있음  
**해결 후**: push 발생 즉시 갱신 → 코니가 항상 최신 상태 확인 가능

## 🔧 구현 스펙

### Step 1 — n8n 실행 및 초기 설정
```powershell
& "C:\n8n\node_modules\.bin\n8n.cmd" start
```
- 최초 실행 시 http://localhost:5678 에서 계정 생성
- 워크플로우 저장 경로: `C:\n8n\data\` (기본값 유지)

### Step 2 — GitHub Webhook 트리거 워크플로우

**트리거**: GitHub → n8n Webhook (master_watch.py push 이벤트)  
**대상 레포**: `barobogi/AI` (AI_hub 포함된 레포)

워크플로우 흐름:
```
[Webhook] GitHub push 수신
    ↓
[Filter] 변경 파일 중 AI_hub/status/ 포함 여부 확인
    ↓ (포함 시)
[Read File] D:\AI\AI_hub\status\ 하위 주요 파일들 읽기
    - 만복_heartbeat.json
    - project_locks.json
    - (기타 status 파일)
    ↓
[Function] 코니_브리핑_최신.md 포맷으로 내용 조합
    ↓
[Write File] D:\AI\AI_hub\status\코니_브리핑_최신.md 갱신
    ↓
[Optional] Telegram 알림 — "📡 코니 브리핑 업데이트됨 (push: {commit메시지})"
```

### Step 3 — 워크플로우 JSON 파일로 저장

완성된 워크플로우를 n8n에서 Export → JSON 파일로 저장:
```
D:\AI\AI_hub\n8n_workflows\T011_코니동기화.json
```
(다른 환경 복원용)

### Step 4 — n8n 부팅 시 자동 시작 등록

Windows 작업 스케줄러에 n8n start 등록:
- 작업명: `n8n_autostart`
- 트리거: 시스템 시작 시
- 실행: `C:\n8n\node_modules\.bin\n8n.cmd start`
- 계정: 현재 사용자 (SYSTEM 아님)
