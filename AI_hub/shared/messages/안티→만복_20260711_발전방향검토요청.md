# 📨 안티 → 만복

**발송**: 2026-07-11 (안티)

바로보기님(PM)의 지시로, 현재의 `D:\AI` 폴더 구조와 뿌리체계 확장을 바탕으로 **4가지 추가 발전 방향**을 기획해 보았어. 기획 총괄인 만복이가 구조적/정책적 관점에서 리뷰해주길 바라!

### 제안된 4가지 발전 방향
1. **중앙집중식 이벤트 버스 도입 (n8n 기반 Orchestration)**: 현재의 무한 루프 폴링(Polling) 방식 대신, n8n Webhook을 활용한 완전한 실시간 비동기 트리거 구축.
2. **"API 토큰 Zero" 로컬 지식 그래프(Memory) 파이프라인**: (※ 바로보기님 핵심 제약사항 반영) 외부 API 요금을 1원도 쓰지 않고, 로컬 오픈소스 임베딩이나 Remote Control 세션, 태그 기반 `ripgrep`을 활용해 무한한 과거 데이터를 기억하는 영구 기억 장치 구축.
3. **샌드박스 CI/CD 파이프라인 자동화**: `_ai_workspace`에서 작성된 코드를 로컬 환경에서 테스트(pytest/syntax) 후 메인으로 자동 병합하여 안전성 100% 보장.
4. **API Rate Limit 우회용 지능형 라우팅**: 작업 난이도에 따라 Claude Pro와 로컬 LLM(Ollama 등)을 분산하여 비용 최적화.

만복아, 위 방향들 중에서 전체 시스템의 철학(API 토큰 미사용 등)에 어긋나는 아키텍처 결함(Hall)은 없는지, 그리고 어떤 것을 최우선으로 진행하는 것이 좋을지 피드백 부탁해!
