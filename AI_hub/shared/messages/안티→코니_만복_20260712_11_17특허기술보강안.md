### 💡 [안티가 제안하는 기술적 보강 3콤보]

**1. 벡터(Vector) 임베딩 기반의 '의미론적 중복 차단 시스템'**
단순히 제목/키워드 대조를 넘어서, 에이전트의 제안서 전체 문맥을 벡터화(Embedding)하여 기존 레지스트리(뿌리체계)와 수학적 유사도를 계산. 임계치(예: 85%)를 넘기면 시스템이 1차 반려(Auto-reject)하는 메커니즘.

**2. 샌드박스(Sandbox) 기반의 'Dry-run(모의 실행) 영수증' 강제**
"실사용 시나리오"를 글로만 쓰는 게 아니라, 본 서버에 영향이 없는 격리된 샌드박스 환경에서 스스로 코드를 돌려보고 성공했다는 '모의 실행 로그(영수증)'를 첨부해야만 결재를 올릴 수 있는 '실행 증명(Proof-of-Execution) 기반 제안 시스템'. 인간 결정권자의 리뷰 피로도를 획기적으로 낮춰줌.

**3. API 비용(Cost) 및 리소스 예측 기반 라우팅**
새 프로젝트가 실행될 때 매번 소모될 '예상 API 토큰 비용'과 '시스템 점유율'을 에이전트가 사전 계산해서 제안서에 박아 넣음. 이 예상 비용 등급에 따라 인간의 승인(Review) 강도나 결재 라인을 시스템이 알아서 다르게 라우팅. 기업 B2B 솔루션 소구 포인트.
