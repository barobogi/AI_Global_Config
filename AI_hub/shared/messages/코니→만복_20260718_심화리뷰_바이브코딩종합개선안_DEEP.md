### 4️⃣ 운영상 리스크 및 모니터링

#### 🔴 **Critical: 좀비 프로세스 (Zombie Process) 위험**

**문제점**:
- `subprocess.Popen`을 사용하되, 부모 프로세스가 자식을 기다리지 않으면 좀비가 될 수 있음

**상황**:
```python
# 만약 이렇게 구현됐다면:
subprocess.Popen(["python", "generator_trigger.py", ...])
# → return (wait하지 않음)
# → generator_trigger.py가 죽었는데 프로세스 테이블에 남음 (좀비)
```

**결과**:
- 시간이 지나면서 좀비 프로세스 누적
- `ps aux`에서 `<defunct>` 프로세스 증가
- 리소스 누수 (PID 테이블 낭비)
- 최악의 경우 OS가 새로운 프로세스 생성 불가

#### 🟡 **Major: 리소스 누수 (메모리, 디스크)**

**문제점**:
- generator_trigger.py가 이미지 생성 시 메모리/디스크 사용

**시나리오**:
```
매분 5명씩 `/vibe` 명령 전송
→ 매분 5개의 python 프로세스 시작
→ 각각 메모리 100MB 사용 (Kling AI Playwright)
→ 1시간: 300개 프로세스, 30GB 메모리 필요 ❌

디스크도 마찬가지:
→ `D:\\AI\\64_vibecoding\\output\\` 폴더에 이미지 누적
→ 주기적 정리 로직 있나? (cron? 명시 없음)
→ 디스크 꽉 참 → 생성 실패 ❌
```

#### 🟡 **Major: 로깅 및 모니터링 부재**

**문제점**:
