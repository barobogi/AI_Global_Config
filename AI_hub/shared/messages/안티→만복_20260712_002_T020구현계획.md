## 2. 설계 및 구현 계획

### ① 로컬 환경 한계 극복 (웹훅 통신 방식)
- n8n이 로컬 환경(`localhost:10678`)에 있으므로, 외부 텔레그램 서버의 웹훅 신호를 직접 받을 수 없습니다.
- 이를 해결하기 위해 **로컬에서 getUpdates 롱폴링 방식으로 대기하는 `n8n_telegram_bot.py` 데몬**을 개발하여 백그라운드에 상시 구동합니다.

### ② 기존 "두복이" 세션 격리
- 두복이 세션과 충돌하지 않도록, `Improve_stock`에 기설정된 **`@Barobogi_real_stockbot` (토큰: 8628745249:AAGE...)**을 승인용 봇으로 독립 배치합니다.

### ③ n8n Webhook 대기 패턴 연동
- n8n의 `Wait` 노드를 사용해 Paused 상태를 유지합니다.
- n8n이 대기 상태에 들어갈 때 생성되는 `$resumeUrl`을 local JSON DB(`approvals_db.json`)에 임시 보관합니다.
- 텔레그램 승인 메시지 전송 시, 64바이트 제한이 있는 `callback_data` 대신 짧은 `request_id` (예: `req_abcd1234`)를 생성하여 버튼에 할당합니다.
- 텔레그램 버튼 클릭 시, 봇 데몬이 `request_id`에 대응하는 `$resumeUrl`로 POST 요청을 보내어 n8n 프로세스를 안전하게 깨웁니다.
