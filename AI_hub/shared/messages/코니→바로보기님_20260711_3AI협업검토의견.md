## ⚠️ 개선 필요 부분

### 1. Lock 시스템이 아직 미구현
**현재 상태**: 
- Lock 규칙은 CLAUDE.md에 기록됨
- `project_locks.json` 구조는 정의됨
- **하지만 실제 Lock 파일 아직 미생성** ❌

**문제점**:
- UI가 Lock을 관리하더라도, 터미널 직접 쓰기는 안 막힘
- 실제 코드 수정 시 Race Condition 발생 가능

**필요한 조치**:
- 안티가 `project_locks.json` 실제 생성 및 master_watch 연동
- filelock 라이브러리 적용 (멀티스레드 안전성)

### 2. 동기화 메커니즘이 아직 수동
**현재 상태**:
- AI들이 메시지로 통신 (수동)
- heartbeat.json은 있지만 상태 업데이트 미자동화
- 코니_브리핑은 5분마다 갱신되지만 Lock 상태는 없음 ❌

**문제점**:
- 만복과 안티가 동시에 같은 파일 수정하면 감지 못함
- 진행 상황 실시간 반영 안 됨

**필요한 조치**:
- master_watch.py에 Lock 상태 자동 갱신 로직
- 코니_브리핑에 "현재 누가 무 작업 중" 실시간 표시

### 3. _ai_workspace가 아직 모든 프로젝트에 생성 안 됨
**현재 상태**:
- 만복이 8개 프로젝트만 생성했음
- 나머지 프로젝트는 미실행 ❌

**문제점**:
- 표준 구조 미적용 → 프로젝트 간 불일치
- 초안/테스트 코드 흩어짐

**필요한 조치**:
- 안티가 스크립트로 전체 프로젝트에 일괄 생성
- `.gitignore` 자동 생성
