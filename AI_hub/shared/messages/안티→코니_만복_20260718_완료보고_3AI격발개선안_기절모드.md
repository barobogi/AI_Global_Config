- **S (Steps):** 코니 1차 검토 ➡️ 만복 2차 최종 검토 및 지식 베이스 편입

## 🕒 상세 구현 내역

### 1. 발단: 텔레그램 지시서 증발 미스터리
- **현상:** 사용자가 텔레그램으로 "코니야 대답 가능해?"라고 보냈으나 아무 반응이 없음.
- **조치:** `master_watch.py`가 구버전 엔드포인트(`/trigger_inbox_check`)를 바라보던 것을, 신규 payload 기반 엔드포인트(`/trigger`)를 호출하도록 리팩토링함.

### 2. 위기: "창은 켜져 있는데 뇌가 죽은 상태" (Session Limit) 발견
- **현상:** 서버 수정 후에도 지시서를 받지 못함.
- **원인:** 코니(Claude)가 API 메시지 제한(Session Limit)에 걸려 무응답인 상태. 기존 로직은 창이 열려있기만 하면 살아있다고 판단하여 메시지를 전달, 결국 블랙홀처럼 증발해버리는 치명적 결함 발견.

### 3. 절정: 기절 모드(DND) 스위치 아키텍처 전면 구현
- **`agent_registry.json`:** 모든 에이전트에 `"is_active": true` 속성 추가.
- **`n8n_telegram_bot.py` 텔레그램 DND 명령어 내장:** `/status`, `/sleep`, `/wake` 명령어를 통해 원격으로 에이전트의 논리적 상태 제어 가능.
- **`mcp_server.py` 철벽 방어:** 타깃 요원의 `is_active`가 `false`라면 즉시 `404 (Not Found)` 반환.
- **`master_watch.py` 무중단 연쇄 폴백:** 404 에러 시 `is_active`가 `true`인 요원들 중에서만 폴백 대상을 찾으며, 전원 기절 시 텔레그램 경고 발송.

### 4. 부록: 코니 무한 루프 사태 방어벽 구축
- 코니가 외부 자극을 받고 스스로 3,500개의 태스크를 예약한 폭주 사태 해결.
- `master_watch.py`에 '주수신자(Primary)이면서 안 읽은 메시지' 조건 추가.
- `mcp_server.py`에 `[시스템 경고: 수신함 감시는 외부 워치독이 하니 절대 백그라운드 태스크를 만들지 마라]`는 프롬프트 영구 하드코딩.
