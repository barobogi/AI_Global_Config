## 1. 프로젝트 개요
* **출처**: 동영상 뽀개기 1번 (skKPHDdinqk - 당근마켓 견적 이야기)
* **목표**: 코딩 지식 없이 자연어/음성 기획안만으로 즉시 작동하는 단일 페이지 웹앱(SPA)을 생성하는 파이프라인 구축
* **뿌리체계 편입 위치**: `64_VibeApps` (또는 24_도구뿌리 산하)

## 2. 도입 배경 및 필요성
바로보기님이 매번 안티에게 "A 파일 열어서 B 줄 수정해 줘"라고 지시하는 대신, "이런이런 기능이 있는 독립된 계산기 앱 만들어줘"라고 지시하면, 안티가 기획부터 디자인, 코드 작성, 로컬 배포까지 한 번에 끝내는 도구가 필요해.

## 3. 핵심 기능 동작 시나리오 (Human-in-the-loop)
1. **트리거 (모바일 앱 입력 환경)**
   * 입력 도구를 데스크탑 채팅창에 한정하지 않고 **모바일 앱(APK)** 환경으로 확장.
   * **1단계**: 현재 모바일에서 가장 접근성이 좋은 3AI 텔레그램 봇을 모바일 입력기로 활용해 기획안 전송.
   * **2단계**: 추후 본 파이프라인 자체를 이용해 'VibeCoding 전용 입력기 APK'를 자동으로 생성하여 스마트폰에 설치하는 방향 추진.
2. **AI 자율 생성 (Vibe Generation)**
   * 안티가 뿌리체계 규칙에 맞춰 `D:\AI\64_VibeApps\[프로젝트명]` 폴더를 자동 생성.
   * 최신 모던 웹 디자인(Tailwind CSS, Glassmorphism 등)을 적용하여 `index.html`, `app.js`, `style.css` 작성 및 저장.
3. **로컬 샌드박스 배포 & 모바일 확인**
   * 웹앱 생성이 완료되면 텔레그램이나 모바일 앱을 통해 결과물을 확인할 수 있는 링크 반환.
4. **1:1 피드백 루프**
   * 바로보기님이 모바일로 결과물을 확인하고 "색상을 어둡게 해줘"라고 피드백하면 안티가 즉시 코드 수정.

## 4. 필요한 자원 및 구현 방식
* **필요 스크립트**: 모바일 입력(텔레그램 봇 연동 또는 전용 API 서버)을 파싱하여 안티그래비티로 전달하는 브릿지 스크립트 필요.
* **디자인 템플릿 제약**: 퀄리티 보장을 위해 반드시 'Modern UI, Dark Mode 지원, Tailwind CSS 사용, 모바일 반응형 완벽 지원'을 프롬프트에 내재화.
