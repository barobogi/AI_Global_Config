## 📌 안티의 제안 및 질의사항

**1. `youtube_pobbagi.py` 스크립트 개발 범위**
- **현황**: `tasks.json` 상에는 명시되어 있으나, 현재 해당 스크립트의 실체가 없는 것으로 보입니다.
- **안티 제안**: 이번 T018 작업에 `youtube_pobbagi.py` 뼈대 개발까지 포함하여 제가 신규로 작성하겠습니다. n8n `Execute Command` 노드에서 `--url "유튜브주소"` 형태로 찔러주면 작동하도록 `argparse` 구조를 씌우는 것이 가장 깔끔할 것 같습니다. (뿌리체계 위치 제안: 24 도구뿌리 또는 41 스크립트)

**2. Firebase 개념노트 연동 주체 (Python vs n8n)**
- **옵션 A (Python 캡슐화)**: 파이썬 스크립트 내부에서 `firebase-admin` 라이브러리를 써서 뽀개기 결과물을 직접 DB에 쏘는 방식.
- **옵션 B (n8n HTTP 노드)**: 파이썬은 순수하게 요약(학습카드 생성)만 해서 JSON으로 뱉어내고, n8n의 HTTP Request 노드가 그 JSON을 받아서 Firebase REST API로 쏘는 방식.
- **안티 제안 (옵션 A 추천)**: n8n 노드가 너무 복잡해지는 것을 막고, 추후 코니가 독립적으로 이 스크립트를 단독 실행할 수도 있으므로 파이썬 코드 내에서 Firebase 통신까지 끝내는 '캡슐화(옵션 A)' 방식을 추천합니다.
