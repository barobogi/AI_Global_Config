## 실증 사례 (오늘 만복+바로보기님 직접 경험)

### 사건 1: BytePlus Sideo 크레딧 확인 불가
```
상황: Sideo(BytePlus) 비디오 생성 테스트 전 무료 크레딧 확인 시도
과정: Console 접속 → Billing/Credits 메뉴 탐색
결과: 잔여 크레딧 확인 불가 (UI 복잡, 위치 불명확)
→ 상업 이용 가능 여부 판단 불가 → Kling AI로 대체
```

### 사건 2: Kling AI 약관 접근 차단
```
상황: Kling AI로 생성한 배경 영상(720×1280, 5초)을 유튜브 업로드 전 상업 이용 허가 확인 시도
과정: 만복이 klingai.com/terms-of-service, /privacy-policy, /faq 직접 접근
결과: 모두 HTTP 446 Unknown Status — 외부 접근 차단
→ 약관 내용 확인 불가 → "직접 확인해야 한다"고 바로보기님께 안내
```

### 사건 3: 수동 확인 지시 (현재 상태)
```
바로보기님이 직접 사이트 접속해서 확인하도록 요청
→ 아직 미확인 상태 (어떤 플랜이 상업 이용 가능한지 불명확)
```

**이 경험이 증명하는 것**:
- 실제 사용자(바로보기님, 만복)가 약관 확인에 실패
- 생성 플랫폼(Kling, BytePlus)이 약관 정보를 쉽게 노출하지 않음
- 수동 확인에 시간 소요 + 실패 가능성 높음
→ **자동 판단 시스템의 필요성 실증**
