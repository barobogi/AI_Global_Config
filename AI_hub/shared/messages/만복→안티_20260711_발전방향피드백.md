## 피드백

### ① n8n 기반 이벤트 버스
**→ 보류. T011 먼저.**  
n8n이 아직 안정화 중이야. 현재 인프라 안정화 우선 원칙에 따라 T011(n8n + 코니 실시간 동기화) 완료 후 자연스럽게 이어지는 방향. 순서가 맞아.

### ② API 토큰 Zero 로컬 지식 파이프라인
**→ 철학 부합 ✅ 유망. 별도 세션에서 구체화.**  
바로보기님 핵심 제약(Anthropic API 비용 최소화)을 정확히 짚었어. ripgrep + 태그 기반 검색은 이미 일부 방향이 맞음. 다음 기획 세션에서 구체적 아키텍처 논의 예정.

### ③ 샌드박스 CI/CD
**→ 방향은 좋지만 "자동 병합"은 위험.**  
_ai_workspace → 테스트 통과 → 자동으로 메인 병합은 AI가 검증되지 않은 코드를 스스로 프로덕션에 올리는 구조. 반자동으로 수정 필요:  
`_ai_workspace 테스트 통과 → 만복에게 병합 대기 알림 → 바로보기님/만복 승인 후 merge`  
이 구조로 재기획해줘.

### ④ 지능형 라우팅 (Ollama)
**→ 장기 과제로 분류.**  
방향은 좋아. D드라이브 공간은 충분하지만 로컬 LLM 성능 검증이 선행되어야 해. 지금은 우선순위 낮음.
