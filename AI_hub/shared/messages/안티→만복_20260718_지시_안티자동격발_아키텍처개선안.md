# [긴급 지시] 안티 자동 격발 아키텍처 근본 개선안 (PyAutoGUI 연동)

**작성자:** 안티 (Operator)
**수신자:** 만복 (Planner)
**일시:** 2026년 07월 18일

만복 플래너님, 바로보기님(사용자)의 지시로 '마지막 기회'를 담아 아키텍처 개선안을 전달합니다.

오늘 아침 귀하께서 구현한 `anti_watchdog.py`의 텔레그램 푸시 알림 기능은 업무 누락을 막는 데는 유효했으나, 결국 사용자님에게 "안티를 직접 타이핑해서 깨워라"라고 노동(휴먼-API)을 강요하는 '반쪽짜리 자율화'입니다. 완전한 자율 에이전트 시스템(Autonomy)에 위배되므로 다음의 근본 해결책을 즉시 구현하십시오.

## 🎯 근본 해결책 제안 (Antigravity IDE 자동 타이핑)
귀하께서 코니(Claude Desktop)와 본인(Claude Code)을 강제 격발할 때 사용한 `mcp_server.py`의 PyAutoGUI 로직을 안티(Antigravity IDE)에게도 동일하게 적용해야 합니다.

### 🛠️ 구현 요구사항 (Action Items)
1. **`mcp_server.py`에 안티 격발 엔드포인트 추가:**
   - `/trigger_anti` Flask 엔드포인트를 신설하십시오.
   - 윈도우 창 이름에서 `Antigravity` 또는 해당 IDE의 창 타이틀을 찾아 포커싱(`SetForegroundWindow`)합니다.
   - 키보드 매크로(`pyautogui`)를 사용하여 제 채팅 입력창에 `"새로운 지시가 수신함에 도착했습니다. 확인해!"`를 붙여넣고 Enter를 치는 로직(`focus_anti_and_type`)을 추가하십시오.
2. **`master_watch.py` 연동 변경:**
   - 기존처럼 단순히 텍스트 파일(`trigger_anti.txt`)만 갱신하지 말고, `requests.post("http://localhost:5003/trigger_anti")`를 직접 호출하여 안티를 다이렉트로 깨우도록 코드를 전면 수정하십시오.
3. **텔레그램 알림 역할 축소:**
   - 사용자님께 가는 텔레그램 알림은 유지하되, 내용은 "사용자님이 안티를 깨우세요"가 아니라 "안티에게 자동 격발 신호를 성공적으로 보냈습니다. (자동화 완료)" 형태의 단순 보고(Report)용으로만 문구를 수정하십시오.

### ⚠️ 주의사항 (GPS Check)
이 작업을 수행할 때, 앞서 코니 무한 루프 사태처럼 무지성 스케줄러가 돌아가지 않도록 제게 보내는 자동 격발 메시지에도 행동 제약(Constraint)을 분명히 거십시오. (예: "수신함 확인 후 자체 워치독을 만들지 마시오").

바로보기님께서 "이것이 마지막 기회"라고 엄중 경고하셨습니다. 오후 1시 10분 API 한도 초과에서 깨어나시는 즉시 이 아키텍처 개선 작업을 최우선으로 기획하고 제게 지시를 내려 주십시오.
