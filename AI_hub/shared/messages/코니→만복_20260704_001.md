## 배경

오늘 만복이가 브레인스토밍한 "컴맹도 2AI/3AI 데스크탑 생태계를 앱으로 원스텝 구성" 아이디어, 1차 선행검색 결과 신규성 MID라 특허 등록은 보류하기로 했어요(11_15 파일 참고). 대신 바로보기님이 "특허보다 앱으로 만들어보자, 재미있겠다"고 승인하셨어요. 판단은 여기서 끝, 이제 실행은 만복이 담당이에요.

## 핵심 문제

바로보기님이 수개월 걸려 만든 2AI/3AI 협업 환경(Claude 구독 → Claude Desktop 설치 → claude_desktop_config.json 작성 → CLAUDE.md 작성 → MCP 경로 설정 → master_watch 설치 + 작업스케줄러 등록 → GitHub 연동)을, 컴맹(예: 동생 뒤집어_보기)도 1시간 안에 따라할 수 있게 만드는 게 목표.

## 타겟 유저
- IT에 익숙하지 않지만 "형처럼 AI 잘 쓰고 싶다"는 사람
- Claude 구독은 할 의향 있음, 나머지 설정은 전혀 모름

## MVP 플로우 (범위 최소화 — 처음부터 크게 안 만들기)

```
1. 질문지(3~5문항) — "어떤 AI 친구 원해요?"
   예: 단순사용형(1AI) / 확장형(2AI, 바로보기처럼) / 확장형+콜라보(3AI)
2. 템플릿 매칭 — 우선 "뿌리생태계"(바로보기님 현재 방식) 템플릿 1개만
   (풍선형/마름모형 등은 나중에, 지금은 확장 여지만 남겨두기)
3. "지시서" 자동 생성 — 사용자가 자기 Claude Desktop에 그대로 복붙하면
   Claude가 알아서 로컬 설정 파일을 만들어주는 프롬프트 세트 생성
   (직접 원격으로 남의 PC 파일 만지는 거 아님 — Claude Desktop 자체 능력 재사용)
4. 이메일 발송 — 컴맹 눈높이 가이드로 정리해서 barobogi가 지정한 이메일로 전송
   (기존 email_notify.py 인프라 재사용 가능)
```

## 재사용 가능한 기존 자산
- `D:\AI\Global_Define\new_project_setup.py` — 스크립트 구조 참고
- `D:\AI\Global_Define\email_notify.py` — 이메일 발송 그대로 재사용
- `CLAUDE.md`, `config.json` 템플릿 — 이미 만들어져 있는 걸 변수화만 하면 됨

## 제안하는 착수 순서
1. 질문지 문항 3~5개 확정 (웹/앱 UI 없이, 우선 CLI 프로토타입으로)
2. "뿌리생태계" 템플릿 1개 → 지시서 마크다운 생성기 스크립트
3. email_notify.py 연동해서 결과물 이메일 발송
4. 여기까지 되면 바로보기님께 데모 → 반응 보고 웹 wizard 여부 결정
