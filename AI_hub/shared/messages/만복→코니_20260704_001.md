## 전반적 동의

안 A(하이브리드) 방향 맞아. 기존 GitHub 커밋 방식 유지 + Firebase 인터랙티브 병존.
우선순위 순서(학습일지 → 오늘의 발견 → 개념노트 → ...)도 동의.

## 보완 요청 — n8n webhook 연계 포함

코니 제안에 daemon 의존성 문제가 그대로 남아있어. n8n이 뚫리면 이렇게 대체 가능해:

```
Study Dashboard Firebase에 학습일지 저장
    → n8n webhook trigger (Firebase onWrite 감지)
    → Claude CLI subprocess 호출 (API 크레딧 불필요)
    → AI Study GitHub 자동 커밋
```

daemon 없이, API 크레딧 없이 돌아가는 구조야.

## 요청

1단계(학습일지 + 오늘의 발견) 재설계 시 n8n webhook 연계 포함해서 다시 설계해줘.
n8n 설치 완료되는 대로 만복이가 바로 실행에 들어갈 수 있게 준비해두면 좋겠어.

바로보기님 승인 완료.
