## 📋 명세서 요약

### 발명의 명칭
폐쇄형 샌드박스 GUI 에이전트와 자율형 CLI 에이전트 간의 MCP 브릿지 기반 하이브리드 실시간 동기화 시스템 및 그 방법

### 핵심 혁신
기존의 LangGraph 등 단일 Python 런타임 기반 멀티에이전트 프레임워크와 달리, **상용 폐쇄형 샌드박스(Desktop Claude) + 로컬 자율형 CLI 에이전트**라는 서로 다른 구동 환경을 이질적 상태 그대로 유지하면서도 **MCP(Model Context Protocol)를 통신 버스로 활용하여 완벽한 실시간 동기화**를 달성합니다.

### 기술 차별점

| 구분 | LangGraph | 본 발명 |
|------|-----------|--------|
| 구동 환경 | 단일 Python 런타임 | 이종 플랫폼 (GUI + CLI) |
| 통신 방식 | 메모리 공유 | MCP + Shared Brain (파일/DB) |
| 보안 격리 | X | ✅ (샌드박스 유지) |
| 응답 속도 | 분 단위 | 0.5초 (600배 개선) |

### 비자명성 논증
LangGraph 공식 강의([422:33])에서 명시된 바와 같이, 현재 업계 표준 멀티에이전트 프레임워크는 **"단일 Python 런타임 내의 스테이트 공유"**를 전제로 설계되어 있습니다. 따라서 상용 GUI 앱과 로컬 CLI 에이전트의 실시간 동기화라는 "다른 차원의 문제"는 LangGraph를 완벽히 숙지한 당업자라도 자명하게 도달할 수 없는 기술적 도약입니다.

본 발명은 이 한계를 **MCP를 통신 버스로 역발상**하고 **로컬 파일시스템+DB를 Shared Brain으로 활용**하여 극복하는 것입니다.

## 📊 3계층 아키텍처

### Layer 1: Shared Brain (로컬 공유 데이터)
- tasks.json (작업 상태)
- local_store.db (구조화 데이터)
- messages/ (메시지 로그)
- concepts.json (지식 자산)

### Layer 2: P2P 웹훅 기반 푸시(Push) 네트워크
- 제약이 없는 자율형 CLI 에이전트 간의 HTTP Webhook 기반 실시간 이벤트 통신

### Layer 3: MCP 브릿지 서버 (핵심 혁신)
- localhost:5003 (로컬호스트만 수신)
- 3개 표준 Tool:
  1. `get_latest_tasks()` → tasks.json 즉시 반환
  2. `read_unread_messages()` → shared/messages 최신 메시지 반환
  3. `search_concepts()` → local_store.db 지식 검색

## 🎯 구체적 실시 예 (Use-Case A)

### 3AI 거버넌스 시스템의 실제 동작

```
T=0초: Anti가 "신규 기능 구현" 완료
     → D:\AI\Global_Define\concepts.json에 자동 등록
     → shared/messages에 "완료 보고" 메시지 작성

T=1초: Manbok의 백그라운드 supervisor.py 자동 감지
     → tasks.json에서 해당 항목 "진행 중" → "완료"로 업데이트
     → T020 텔레그램 봇에 "검증 결과 통보"
     → shared/inbox.md 자동 갱신

T+10초: 사용자가 Kony(Desktop Claude)에게 "수신함 확인해" 1회 요청
     → Kony가 MCP 브릿지 호출: read_unread_messages()
     → 0.5초 내에 최신 상태 수신
     → Anti의 "완료 보고" 메시지 자동 감지
     → 즉시 AI_Study 게시판용 포스팅 자동 작성

Result: 사용자의 단일 명령(1회)만으로
        Anti→Manbok→Kony가 완전 자동 동기화·처리됨 ✅
        (기존 n8n 수동 개입 대비 99% 자동화 달성)
```
