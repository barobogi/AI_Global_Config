## 🎯 비자명성(Non-obviousness) 논증 구조

특허법상 비자명성은 이렇게 증명해:
> "해당 기술 분야의 **통상의 기술자**(당업자)가 선행 기술들을 결합하더라도
> 본 발명에 자명하게 도달할 수 없어야 한다."

우리의 논증 흐름은 이렇게 써줘:

```
[Step 1] 당업자의 수준 = LangGraph 숙련자로 정의
  → "LangGraph는 현재 AI 에이전트 개발의 사실상 표준(de facto standard)
     프레임워크로, 본 발명의 출원 시점 기준 가장 앞선 멀티에이전트
     오케스트레이션 기술을 대표한다."

[Step 2] 당업자(LangGraph 숙련자)가 알고 있는 최대치
  → LangGraph 공식 강의(블루닷 AI, 9시간 풀강의)에서 직접 언급:
     "지금까지 우리가 구현했던 에이전트는 단일 에이전트였습니다.
      단 하나의 그래프만 있었죠." — [422:33]
  → LangGraph의 가장 발전된 형태인 멀티에이전트/서브그래프조차
     단일 Python 런타임 내에서만 동작함이 공개 강의에서 명시됨.

[Step 3] 당업자가 도달할 수 없는 지점 = 본 발명
  → LangGraph 아키텍처는 설계 전제 자체가 "동일한 실행 환경"이므로,
     상용 GUI 앱(Desktop Claude 등 샌드박스 환경)과의 이종 플랫폼 동기화는
     LangGraph의 기술적 발상 범위 밖에 있음.
  → 따라서 LangGraph를 완전히 숙지한 당업자라도
     MCP 브릿지를 통한 이종 플랫폼 하이브리드 동기화에 자명하게 도달 불가.

[결론] 비자명성 ✅ 확립
```
