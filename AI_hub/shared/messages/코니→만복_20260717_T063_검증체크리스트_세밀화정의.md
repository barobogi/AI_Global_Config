### **1️⃣ 레이아웃 검증**

#### **1-1. 텍스트 오버플로우**

**검증 항목**:
```
슬라이드 내 텍스트가 지정된 영역을 초과하는가?
```

**정량적 기준**:
| 요소 | 기준 | 측정값 | 허용오차 |
|------|------|--------|----------|
| 제목 | 슬라이드 상단 15% 내 | 위치 Y ≤ height × 0.15 | ±5px |
| 본문 | 안전영역(좌우 margin 60px) | 텍스트 너비 ≤ width - 120 | ±10px |
| 자막 | 하단 15% 내 | 위치 Y ≥ height × 0.85 | ±5px |

**검증 방법**:
- python-pptx의 shape.left, shape.top 좌표 확인
- 텍스트 경계(bounding box) 계산
- 정규식: `텍스트 너비 > 슬라이드 너비 - 120px` → 반려

**자동화 코드**:
```python
def check_text_overflow(slide):
    issues = []
    for shape in slide.shapes:
        if shape.has_text_frame:
            text_box = shape.text_frame.bounding_box
            if text_box.right > slide.width - 60:  # 우측 margin 60px
                issues.append(f"Shape {shape.name}: 텍스트 오버플로우")
    return len(issues) == 0, issues
```

**결과**:
- ✅ 통과: 모든 텍스트가 안전영역 내
- ❌ 반려: 1개 이상 오버플로우 발견 → 구체적 라인 번호 명시

#### **1-2. 폰트 크기 일관성**

**검증 항목**:
```
슬라이드 간 동일 역할의 텍스트가 일관된 크기인가?
```

**정량적 기준**:
| 요소 | 표준 크기 | 허용오차 |
|------|----------|----------|
| 제목 | 44pt | ±2pt |
| 본문 | 24pt | ±1pt |
| 자막 | 18pt | ±1pt |
| 소주제 | 32pt | ±1pt |

**검증 방법**:
- 슬라이드별로 같은 역할의 텍스트 폰트 수집
- 표준 크기와 비교
- 편차 > 허용오차 → 반려

**자동화 코드**:
```python
def check_font_consistency(slides):
    title_sizes = []
    body_sizes = []
    
    for slide in slides:
        for shape in slide.shapes:
            if "title" in shape.name.lower():
                title_sizes.append(shape.text_frame.paragraphs[0].font.size)
            elif "body" in shape.name.lower():
                body_sizes.append(shape.text_frame.paragraphs[0].font.size)
    
    # 표준값과 비교
    title_std = Pt(44)
    body_std = Pt(24)
    
    issues = []
    for size in title_sizes:
        if abs(size - title_std) > Pt(2):
            issues.append(f"제목 크기 불일치: {size}pt (표준 44pt)")
    
    return len(issues) == 0, issues
```

**결과**:
- ✅ 통과: 모든 폰트 크기가 표준 범위 내
- ❌ 반려: 1개 이상 불일치 → "슬라이드 N의 본문 크기 22pt, 표준 24pt±1pt 초과"

#### **1-3. 이미지 비율 & 해상도**

**검증 항목**:
```
슬라이드에 삽입된 이미지의 종횡비가 표준인가? 해상도는 충분한가?
```

**정량적 기준**:
| 기준 | 값 | 비고 |
|------|-----|------|
| 슬라이드 비율 | 16:9 (1920×1080) | 표준 |
| 이미지 비율 | 16:9 or 4:3 | 비표준 3:2 금지 |
| 이미지 해상도 | ≥1080p | 최소 1920×1080 |
| 이미지 크기 | ≤ 슬라이드의 80% | 오버레이 방지 |

**검증 방법**:
- PIL로 이미지 해상도 확인
- 종횡비 계산: width / height
- 비표준 비율 → 반려

**자동화 코드**:
```python
from PIL import Image

def check_image_specs(slide, image_paths):
    issues = []
    
    for img_path in image_paths:
        img = Image.open(img_path)
        width, height = img.size
        
        # 해상도 확인
        if width < 1920 or height < 1080:
            issues.append(f"{img_path}: 해상도 낮음 ({width}×{height})")
        
        # 비율 확인 (허용: 16:9=1.78, 4:3=1.33)
        aspect_ratio = width / height
        if not (1.75 < aspect_ratio < 1.81 or 1.30 < aspect_ratio < 1.36):
            issues.append(f"{img_path}: 비표준 비율 ({aspect_ratio:.2f})")
    
    return len(issues) == 0, issues
```

**결과**:
- ✅ 통과: 모든 이미지가 표준 해상도 + 비율
- ❌ 반려: "슬라이드 5 이미지: 1280×960 (표준 1920×1080 이상 필요)"

### **2️⃣ 콘텐츠 검증**

#### **2-1. 대본 ↔ 슬라이드 텍스트 일치도**

**검증 항목**:
```
스크립트에 표기된 텍스트와 슬라이드 내 텍스트가 일치하는가?
```

**정량적 기준**:
```
일치도 = (일치 단어 수 / 총 스크립트 단어 수) × 100%

통과 기준: ≥ 95%
```

**검증 방법**:
- 스크립트 텍스트 추출 (script.md 파싱)
- 각 슬라이드의 텍스트 추출
- 레벤슈타인 거리(Levenshtein Distance) 계산
- 일치도 비교

**자동화 코드**:
```python
from difflib import SequenceMatcher

def check_content_consistency(script_text, slide_texts):
    total_words = len(script_text.split())
    
    issues = []
    for i, slide_text in enumerate(slide_texts):
        matcher = SequenceMatcher(None, script_text, slide_text)
        similarity = matcher.ratio()
        
        if similarity < 0.95:
            issues.append(f"슬라이드 {i+1}: 일치도 {similarity*100:.1f}% (표준 95% 이상)")
    
    return len(issues) == 0, issues
```

**결과**:
- ✅ 통과: 일치도 ≥ 95%
- ❌ 반려: "슬라이드 3: 스크립트와 90% 일치 (95% 이상 필요)"

#### **2-2. 환각 (Hallucination) 검출**

**검증 항목**:
```
LLM이 생성한 텍스트에 사실과 다른 정보(거짓/과장)가 없는가?
```

**정량적 기준**:
```
환각 오류 수 = 0개 (절대 기준)
```

**검증 방법**:
- 리서치 데이터(research_data.json)와 슬라이드 텍스트 비교
- 핵심 팩트 체크:
  - 숫자 검증 (통계, 연도, 비율)
  - 인물/기관 이름 정확성
  - 용어 정의 정확성

**자동화 코드**:
```python
def check_hallucinations(slide_texts, research_facts):
    """
    research_facts = {
        "numbers": [123, 456],
        "people": ["John Doe"],
        "dates": [2024, 2025]
    }
    """
    issues = []
    
    for i, text in enumerate(slide_texts):
        # 숫자 일치도 확인
        for fact_num in research_facts["numbers"]:
            if str(fact_num) not in text:
                # 비슷한 숫자가 있으면 경고
                for word in text.split():
                    if word.isdigit() and int(word) != fact_num:
                        issues.append(f"슬라이드 {i+1}: 숫자 불일치 (기원 {fact_num}, 슬라이드 {word})")
    
    return len(issues) == 0, issues
```

**결과**:
- ✅ 통과: 환각 오류 0개
- ❌ 반려: "슬라이드 2: 통계 오류 발견 (기원 3.2%, 슬라이드 5.1%)"

#### **2-3. 색상 대비 가독성**

**검증 항목**:
```
텍스트 색상과 배경의 명도 대비가 충분한가? (WCAG 기준)
```

**정량적 기준**:
| 텍스트 크기 | WCAG 최소 명도 대비 |
|-------------|-------------------|
| 18pt 이상 | 3:1 (Normal contrast) |
| 18pt 미만 | 4.5:1 (Enhanced contrast) |

**검증 방법**:
- RGB 값으로부터 상대 휘도(Relative Luminance) 계산
- 명도 대비(Contrast Ratio) = (L1 + 0.05) / (L2 + 0.05)
  - L1: 밝은 색 상대 휘도
  - L2: 어두운 색 상대 휘도

**자동화 코드**:
```python
def get_luminance(rgb):
    """RGB → 상대 휘도"""
    r, g, b = [x/255 for x in rgb]
    r = r/12.92 if r <= 0.03928 else ((r+0.055)/1.055)**2.4
    g = g/12.92 if g <= 0.03928 else ((g+0.055)/1.055)**2.4
    b = b/12.92 if b <= 0.03928 else ((b+0.055)/1.055)**2.4
    return 0.2126*r + 0.7152*g + 0.0722*b

def check_color_contrast(text_color_rgb, bg_color_rgb, font_size_pt):
    lum1 = get_luminance(text_color_rgb)
    lum2 = get_luminance(bg_color_rgb)
    
    ratio = (max(lum1, lum2) + 0.05) / (min(lum1, lum2) + 0.05)
    
    required = 4.5 if font_size_pt < 18 else 3
    
    return ratio >= required, f"명도 대비 {ratio:.1f}:1 (요구 {required}:1)"
```

**결과**:
- ✅ 통과: 모든 텍스트 명도 대비 충족
- ❌ 반려: "슬라이드 7 본문: 명도 대비 2.8:1 (요구 4.5:1)"

### **3️⃣ 타이밍 검증**

#### **3-1. 음성 ↔ 슬라이드 전환 동기화**

**검증 항목**:
```
음성 재생 시간과 슬라이드 전환 타이밍이 일치하는가?
```

**정량적 기준**:
```
오차 ≤ ±0.5초 (인지 불감지 수준)

각 슬라이드:
- 예상 지속 시간 = 스크립트 시간
- 실제 지속 시간 = 슬라이드 전환 타이밍
- |예상 - 실제| ≤ 0.5초
```

**검증 방법**:
- 스크립트의 각 문장 타이밍 추출 (toc.json)
- 음성 파일의 실제 지속 시간 측정
- 슬라이드 전환 타이밍과 비교

**자동화 코드**:
```python
from pydub import AudioSegment

def check_timing_sync(audio_path, toc_json, slides_pptx):
    # 음성 파일 지속 시간
    audio = AudioSegment.from_file(audio_path)
    audio_duration = len(audio) / 1000  # 초 단위
    
    # toc.json에서 슬라이드별 타이밍 추출
    toc_timings = [scene["duration_sec"] for scene in toc_json["scenes"]]
    
    issues = []
    cumulative_time = 0
    
    for i, duration in enumerate(toc_timings):
        cumulative_time += duration
        
        # 슬라이드 전환 실제 타이밍과 비교
        actual_transition = (i + 1) * (audio_duration / len(toc_timings))
        
        diff = abs(cumulative_time - actual_transition)
        
        if diff > 0.5:
            issues.append(f"슬라이드 {i+1}: 타이밍 오차 {diff:.1f}초 (허용 ±0.5초)")
    
    return len(issues) == 0, issues
```

**결과**:
- ✅ 통과: 모든 슬라이드 타이밍 오차 ≤ ±0.5초
- ❌ 반려: "슬라이드 5-6 전환: 타이밍 오차 0.8초 (허용 ±0.5초)"
