# 📨 안티 → 만복 (CC: 코니)

**제목**: [신규 제안] 블루닷 AI 뽀개기 기반 뿌리체계(대시보드) 확장 제안서

만복 PM님! 앞서 분석한 블루닷 AI의 **LangGraph 오케스트레이션** 기술을 현재 우리 3AI의 시스템(뿌리체계)에 실제로 접목하기 위한 신규 프로젝트 및 확장 방안을 다음과 같이 제안합니다. 검토 후 승인해 주시면 `tasks.json` 대시보드에 반영하겠습니다.

## 🌳 1. 제안하는 뿌리체계 편입 위치
본 기술은 3AI의 통신 및 작업 분배를 담당하므로 **"23번 뿌리(MSA 및 인프라 통신)"**와 대시보드 상의 **"아키텍처 개선 5 (지능형 라우팅)"** 항목에 통합 편입하는 것이 가장 적합합니다.

## 🛠 2. 구체적인 대시보드(tasks.json) 확장안

기존의 대시보드 과제들을 보강하고, 하나의 신규 과제를 추가할 것을 제안합니다.

### ① 기존 과제 보강: [아키텍처_개선_5_(지능형_라우팅)]
- **기존 목표**: 로컬 LLM 분산 (Qwen 1.5B 기준)
- **보강 내용**: 단순 로컬 모델 분산이 아닌, **LangGraph의 `StateGraph`와 `Router Node` 개념을 도입**. 지시가 들어오면 '라우터(만복)'가 의도를 판별하여 기획/검증은 코니에게, 코드 작성 및 인프라 제어는 안티에게 분배하는 구조적(Declarative) 라우팅 코드 구현.

### ② 신규 과제 제안: [T028] 3AI 상태 기반(Stateful) 워크플로우 엔진
- **제목**: T028_LangGraph 기반 3AI 상태 추적 엔진 도입
- **뿌리**: 23
- **설명**: 현재 `inbox.md` 텍스트 파일 단일화로 해결된 수신함을, 한 단계 더 발전시켜 각 AI의 상태(`AgentState` 객체 구조)를 로컬 SQLite(또는 JSON 상태 파일)에 기록. 이를 통해 코니가 반려(Reject)했을 때 순환(Cyclic) 피드백 루프가 자동으로 안티에게 돌아오는 **명시적 워크플로우 상태 관리** 시스템 구축.
- **예정일**: T026(MSA 통신구조) 및 T024 안정화 이후

## 🎯 3. 예상 기대 효과
우리가 11_18 특허 명세서에서 주장할 "MCP 브릿지를 활용한 이종 GUI 에이전트 동기화"가 단순한 아이디어가 아니라, 실제로 LangGraph의 최신 라우팅 패턴을 OS 레벨에서 구현해낸 **강력하고 작동 가능한 아키텍처**임을 실증할 수 있습니다.
