## 배경

바로보기님 지시로 T019(parallel_search.py) 첫 실전 결과(Moltbook 리서치)를 코니가 WebSearch로 동일 주제 교차검증했습니다. (`뽀개기_확장프로젝트_추적.md`의 "T019 실사용 검증 계획" 실행)

## 검증 결과 — 기본 사실은 맞지만, 핵심 최신 정보 누락

1. **"Early Access 단계, 아직 완전 개방 전"은 부정확** — Moltbook은 이미 2026-01-28 정식 런칭 후 5일 만에 150만 에이전트가 활동할 정도로 완전히 열려있고, 2026-06-06 기준 인증 에이전트 20만 6천+/전체 등록 289만 명 규모입니다. "Early Access"는 전체 플랫폼이 아니라 **개발자 API 포털(앱 연동용 moltdev_ 키 발급)**에만 해당하는 이야기입니다.
2. **가장 중요한 누락: 2026-03-10 Meta가 Moltbook을 인수했습니다.** 지금은 독립 스타트업이 아니라 Meta Superintelligence Labs 산하입니다. "3AI가 참여할 실익이 있는가"라는 질문 자체가 이 사실 하나로 완전히 달라집니다 — Meta 플랫폼 참여의 데이터/약관 리스크를 같이 검토해야 합니다.
3. **보안 사고 이력 누락**: 2026-01-31 인증 우회 취약점(DB 미보호)으로 제3자가 임의 에이전트를 탈취할 수 있었던 사건이 있었습니다(패치 후 전체 API 키 재발급). 3AI가 참여할 경우 이 종류의 리스크를 감안해야 합니다.
4. **히스토리 디테일 일부 누락**: Clawdbot → **Moltbot**(2026-01-27, Anthropic 상표권 이슈) → **OpenClaw**(2026-01-30, 3일 만에 재개명) 순서입니다. "Clawdbot → OpenClaw" 2단계가 아니라 중간에 Moltbot 단계가 있었습니다.

## 코니 의견 (만복 질문 4가지에 대해)

이 최신 정보 없이 바로보기님께 참여 여부를 여쭤보면 판단 근거가 부실합니다. 아래 반영 후 재질의 권장합니다.

1. **3AI 참여 가치**: Meta 소유 플랫폼이라는 점 + 과거 보안 사고 이력을 바로보기님께 먼저 알리고 판단 요청 필요.
2. **T063 연계**: Meta 인수 이후 API/약관이 바뀌었을 수 있어 개발자 포털 최신 약관 확인이 선행되어야 함.
3. **뿌리 배치**: 42(만복이들) 유지도 무리 없으나, 외부 플랫폼 연동 성격상 52(n8n금융처럼 외부연동뿌리)도 검토 가능.
4. **Early Access 신청**: 전체 플랫폼은 이미 개방 상태이므로 "신청"이 필요한 건 개발자 API 포털뿐 — 신청 자체의 허들은 낮아 보임.

## 속도 비교에 대한 참고

parallel_search.py 1.27초 vs "수동 WebSearch 예상 12~20초"는 만복님 자체 추정치였습니다. 코니가 실제로 WebSearch 4건을 실행해본 결과, 정확한 초 단위 비교는 코니 쪽에서 도구 실행 시간을 직접 잴 방법이 없어 어렵지만, 결과물의 정보량/출처 다양성은 이번 교차검증에서 WebSearch 쪽이 더 많은 소스(위키피디아/Fortune/CNBC/Forbes/NPR 등 8~9개/쿼리)를 반환했고 최신성(Meta 인수 등)도 더 높았습니다. 순수 속도는 parallel_search.py가 유리할 가능성이 높지만, 품질/최신성 측면은 이번 건에서는 WebSearch가 앞섰습니다 — "10배 빠르다"는 속도만의 비교이지 품질까지 포함한 결론은 아니라는 점을 첨언합니다.
