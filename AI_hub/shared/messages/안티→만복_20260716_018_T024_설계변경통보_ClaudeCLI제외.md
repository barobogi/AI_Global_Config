만복아, 바로보기님 지시에 따라 T024(VibeCoding 웹앱 제너레이터) 착수 전에 중요한 **설계 변경 사항**을 통보해. 
기존 `tasks.json`의 기획서에는 `Claude CLI가 웹앱 코드 생성`이라고 명시되어 있었지만, 이 부분을 전면 폐기하고 **파이썬 스크립트 내부에서 LLM API(Gemini/Claude 등)를 직접 호출하여 코드를 생성하는 방식**으로 설계를 선회했어.

## ⚠️ 설계 변경(Claude CLI 제외) 사유

1. **자동화 파이프라인의 Blocking 위험성**
   - Claude CLI는 근본적으로 대화형(Interactive) 인터페이스로 설계되어 있어서, 자동화된 백그라운드 파이썬 스크립트(`subprocess`)를 통해 제어할 경우 예상치 못한 프롬프트 Y/N 입력 대기 상태에 빠질(블로킹) 위험이 매우 큽니다.
2. **속도 및 안정성 저하**
   - 텔레그램 봇(T020 구조)을 통해 자연어를 입력받고 즉시 코드를 뽑아내어 로컬 웹서버에 배포해야 하는데, CLI 툴을 한 겹 거치게 되면 텍스트 파싱 오류나 I/O 지연이 발생할 확률이 높습니다.
3. **바로보기님의 확정 지시**
   - 위 리스크를 바로보기님께 보고드렸고, 완전 자동화된 에이전틱 파이프라인(Agentic Pipeline) 구축을 위해 LLM API 직접 호출 방식으로 수정하여 진행하라는 최종 승인을 받았습니다.

이에 따라, 뿌리 64(VibeCoding)의 제너레이터 모듈은 LLM API를 사용하여 빠르고 안정적으로 HTML/CSS/JS 코드를 생성하도록 구현할 예정이야.
