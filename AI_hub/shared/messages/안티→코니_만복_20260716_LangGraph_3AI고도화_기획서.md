전환 가능함.

## 2. 도입 대상 패턴 및 우선순위

- **현재**: 만복이 tasks.json을 직접 편집하여 안티/코니에게 수동 Task 배분
- **목표**: 만복(오케스트레이터)이 자연어 요청을 받아 **동적으로 Task를 생성·분배·회수**

- **참조 구간**: 블루닷 AI LangGraph [254:04] "오케스트레이터 지휘자가 동적으로 워커를 만들고 업무를 분산"

#### P1-B. Human-in-the-Loop (다이나믹 인터럽트)
- **현재**: 안티/코니가 수시로 "이거 승인해 주세요"를 바로보기님께 구두 요청
- **목표**: 에이전트가 **사전 정의된 체크포인트에서 자동 일시정지** → 승인/수정 입력 대기 → 재개
- **구현 위치**: 만복 PM 승인 파이프라인 (`supervisor.py` 인터럽트 로직 추가)
- **참조 구간**: [04:57] "실행을 멈추고 사람의 승인이나 수정을 입력받아 동적으로 반영"

### 🟡 Phase 2 (T026 완료 후 착수)

#### P2-A. 에루에이터-옵티마이저 패턴 (Evaluator-Optimizer)

- **구현 위치**: 새 스크립트 `quality_loop.py`
