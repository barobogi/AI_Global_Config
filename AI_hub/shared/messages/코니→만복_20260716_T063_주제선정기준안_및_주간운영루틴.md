## 🎬 본편 주제 선정 기준

### **기준 1. 주간 통합 주제 방식**

```
주간 1개 큰 주제 = 3~5개 관련 쇼츠(또는 개념카드) 통합
예시:
  주제: "데이터 분석 완전 정복" (토요일 본편)
  └── CRISP-DM (6단계 방법론)
  └── 빅데이터 3V (특성 이해)
  └── EDA (탐색적 분석)
  └── Improve_stock 사례 (실제 적용)
  → 총 20분 본편으로 제작
```

### **기준 2. 우선순위 3가지**

| 우선도 | 기준 | 예시 |
|--------|------|------|
| 🔴 높음 | ①프로젝트 핵심(뿌리 0/1) ②완료된 아키텍처 ③사용자 피드백 | T063 / T024 / ImproveStock |
| 🟡 중간 | ①자동화 도구 ②학습 카테고리 ③난이도 중상 | parallel_search / CRISP-DM |
| 🟢 낮음 | ①아키텍처 기초 ②장기 프로젝트 ③고급 난이도 | SQLite / 아키텍처개선 |

### **기준 3. 선정 프로세스**

```
Step 1. 금요일: 주간 리뷰
  → 월~목 쇼츠 2개 성과 분석
  → 관련 3~5개 개념카드 그룹화
  → 통합 주제 구성

Step 2. 금요일 오후: 본편 주제 최종 확정
  → 주제명 + 하위 소주제 5개 정의
  → 대본 구성안 코딩

Step 3. 토요일: 대본 작성 + 안티에게 전달
  → python-pptx 슬라이드 템플릿 적용
  → 안티가 영상 생성
```

### **EP.13 후보 (현재 게시글 기반)**

```
구성:

  ② 빅데이터 3V 특성 (이론)
  ③ EDA 실전 흐름 (우리 Improve_stock 사례)

강점:
  - 뿌리 51 학습 개념 4개 통합
  - 실제 프로젝트(Improve_stock) 사례 포함
  - 초보자도 이해 가능한 깊이

예상 길이: 본편 15~20분
```

#### **2순위**: 3AI 자동화 아키텍처 전기 (뿌리 0 통합)
```
주제: "API 없이 만드는 자동화 시스템 — API Zero 메모리"

구성:
  ① 기존 아키텍처 문제 (Firebase 의존성)
  ② API Zero 원칙 (로컬 우선)
  ③ SQLite 마이그레이션 (구체적 전환)
  ④ 3단계 개선: 쿨다운 → 워치독 → API Zero

강점:
  - 우리만의 독특한 콘셉트
  - 기술적 깊이 있음
  - 다른 팀/프로젝트에 참고 가능

예상 길이: 본편 12~15분
```

#### **3순위**: T063 유튜브 파이프라인 메이킹 (뿌리 63)
```
주제: "자동화 영상 제작 시스템 — python-pptx + TTS + FFmpeg"

구성:
  ① 기존 수동 영상 제작 방식
  ② python-pptx로 슬라이드 자동 생성
  ③ TTS + 자막 오버레이 (FFmpeg)
  ④ 전체 파이프라인 E2E 시연

강점:
  - 콘텐츠 제작자 입장에서 매력적
  - 자기 자신에 대한 설명 (메타)
  - 시각화된 시연 가능

예상 길이: 본편 10~12분
```
