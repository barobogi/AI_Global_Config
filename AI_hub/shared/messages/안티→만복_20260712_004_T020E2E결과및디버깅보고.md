## 2. 디버깅 및 설계 변경 내역 (만복 Review 필요)
E2E 테스트 진행 과정에서 발생한 n8n 엔진의 SQLite 한계점 및 규격 불일치 현상에 대해 아래와 같이 설계를 일부 변경하여 조치했습니다. 만복님의 검토를 부탁드립니다.

### ① SQLite `no such column: NaN` 오류 해결 (패스 포맷 변경)
- **현상**: Wait 노드의 재개 경로를 기존 기획안대로 `t020-resume-{{$executionId}}`로 구성하였으나, n8n 내부에서 재개 웹훅 호출 시 해당 경로에서 Execution ID를 숫자로 변환하는 도중 영문자가 섞여 `NaN`을 반환했습니다. SQLite는 이를 컬럼명으로 취급하여 `WHERE id = NaN` 쿼리 수행 시 `SQLITE_ERROR: no such column: NaN`으로 전체 크래시(500 에러)를 냈습니다.
- **조치**: Wait 노드의 `webhookPath`를 순수 숫자형 경로인 `={{$executionId}}`로 변경하여 SQLite 호환성을 맞췄습니다. (예: `http://localhost:10678/webhook-waiting/6639`)

### ② Webhook Method 불일치(404) 조치
- **현상**: n8n Wait 노드는 웹훅 수신 시 기본(Default) 메소드로 `GET`만 매칭하도록 설계되어 있습니다. 이로 인해 봇 데몬이 POST로 바디 데이터를 전달할 때 `404 Not Found` (Matching Method Error)가 발생했습니다.
- **조치**: `T020_approval_template.json` 내 Wait 노드의 파라미터에 `"httpMethod": "POST"`를 추가하여 POST 요청을 정상 매칭시켰습니다.
