### 1. 젠스파크 핵심 기능 3가지

| 기능 | 설명 | 우리 시스템 비교 |
|------|------|----------------|
| **Slides** | 프롬프트 → 리서치 포함 PPT 자동 생성 | ❌ 미구현 (신규 기회) |
| **Web** | 프롬프트 → 웹페이지 즉시 생성·호스팅 | ⏳ T024 VibeCoding (로컬 버전) |
| **Super Agents** | 다단계 병렬 리서치 → 구조화 리포트 | ✅ parallel_search.py (T019 완료) |

**핵심 차별점 (우리 강점)**:
- 젠스파크 = 외부 웹 데이터만 접근 가능
- 우리 = `tasks.json` / `inbox.md` / `heartbeat.json` / `local_store.db` 내부 데이터 직접 활용 가능
- 로컬 파일 수정·자동화 연동 가능

### 2. 즉시 반영 — 개념카드 2개 (뿌리 51)

**개념카드 D: Multi-LLM 오케스트레이션**
```
정의:
  여러 LLM(GPT-4o / Claude / Gemini)을 동시에 실행하고
  각 결과를 비교·취합하여 최적 출력을 생성하는 구조

젠스파크가 SaaS로 구현한 방식 = Super Agents

우리 적용 현황:
  → parallel_search.py (T019): DuckDuckGo + Wikipedia 병렬 검색 ✅ 완료
  → T026 MSA 하이브리드 로깅: 만복↔안티 이벤트 통신 설계 중

확장 방향:
  → T026 완성 후 LLM 라우팅 레이어 추가 시 참고 아키텍처
```

**개념카드 E: 리서치→콘텐츠 원스톱 파이프라인**
```
정의:
  주제 입력 → 자동 리서치 → 구조화(목차·흐름) → 결과물 생성
  을 단일 파이프라인으로 처리하는 설계 패턴

젠스파크 흐름:
  프롬프트 → Super Agents 리서치 → Slides/Web 생성

우리 T063 확장 적용:
  AI Study 주제 → parallel_search.py → 대본 생성 → 슬라이드 생성 → 업로드
  (현재: 대본→TTS→업로드 / 확장: 대본→슬라이드→화면녹화→업로드)
```

→ **뿌리 51 개념노트 DB에 카드 D·E 등록 제안**

### 3. 중기 구현 — T063 슬라이드 자동 생성 모듈 (뿌리 63)

**현재 T063 파이프라인**:
```
대본 작성 → TTS(OpenAI) → 웨이브폼 합성 → YouTube 업로드
```

**젠스파크 인사이트 적용 확장안**:
```
대본 작성
    ↓
[신규] python-pptx로 슬라이드 자동 생성
    ↓
화면 녹화 (슬라이드 + 웨이브폼 오버레이)
    ↓
YouTube 업로드
```

**구현 방법 비교**:

| 방법 | 비용 | 내부 데이터 | 자동화 | 추천 |
|------|------|------------|--------|------|
| 젠스파크 API | 유료 | ❌ | 반자동 | ❌ |
| python-pptx | 무료 | ✅ | ✅ 완전 자동 | ✅ |

→ **python-pptx 자체 구현 권장**
→ T063 2단계 ③시각가이드라인 작업에 슬라이드 자동화 포함 검토
→ 신규 서브 태스크: `T063_슬라이드모듈`
