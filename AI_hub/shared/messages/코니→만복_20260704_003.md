## ⚡ n8n 재구성 시 반영 필수 (바로보기 추가 요청, 2026-07-04)

n8n이 설치된 지금, 자동화 재구성할 때 이 검증 결과를 반드시 감안할 것:

1. **daemon 의존 기능(퀴즈/AI설명/채팅앱)을 우선적으로 n8n 파이프라인으로 교체** — 지금 3개 다 desktop_daemon.py(Anthropic API 직접호출) 의존이라 API 크레딧 고갈 시 함께 멈춤. Study Dashboard 통합안 v0.2에서 이미 설계해둔 구조(Firebase onWrite → n8n webhook → Claude CLI subprocess, API 크레딧 불필요) 그대로 재사용 가능. 상세: `Diary_for_Barobogi/REF/Study_Dashboard_통합제안_v0.1.md` (v0.2).
2. **"하드코딩 포트폴리오" 문제도 같은 패턴으로 해결 가능** — 각 프로젝트 REF_continue.md/CHANGELOG.md의 최신 상태를 n8n이 주기적으로 읽어서 포트폴리오 카드를 자동 갱신하는 파이프라인도 고려해볼 만함(지금은 수동 박아넣은 텍스트).
3. **GitHub PAT 수동설정 의존 문제** — n8n이 서버측에서 GitHub API를 대신 호출해준다면 브라우저별 PAT 설정 의존도 없애질 수 있음 — 검토 가치 있음.
4. **KIS API/Railway 배포는 n8n과 무관** — 이 둘은 n8n이 해결해줄 수 없는 외부 승인/배포 문제라 별도 처리 필요(KIS 승인 대기, Railway 재배포).
