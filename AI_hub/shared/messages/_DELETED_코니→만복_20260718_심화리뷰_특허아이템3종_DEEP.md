## 🔍 Item 1: Cascading Fallback with DND (물리적↔논리적 상태 분리)

### ✅ 기술적 신규성 검증

#### **긍정 평가**:
- **개념의 독창성** ⭐⭐⭐⭐⭐
  - 기존: 창이 열려있으면 "살아있다" 판단 (물리적 상태만)
  - 신규: 창 + API 활성 상태 이중 관리 (논리적 상태 추가)
  - 실제 문제 해결 (좀비 에이전트 감지) ✅

- **DND(Do Not Disturb) 패턴** ⭐⭐⭐⭐
  - 텔레그램 `/sleep`, `/wake` 명령어로 원격 제어
  - 프롬프트 수정 없이 상태 제어 (프로토콜 레벨)
  - 우아한 설계 ✅

- **Cascading Fallback** ⭐⭐⭐⭐⭐
  - 404 에러 시 다음 우선순위 자동 선택
  - "지시서 증발" 문제 근본 해결
  - 무중단 서비스 구현 ✅

#### **우려 사항 (Deep Search)**:

##### 🔴 **Critical: 우선순위 체인이 정적으로 고정되어 있나?**
```

- 동적으로 우선순위를 변경할 수 있나? (e.g., 부하 기반)
- 특정 에이전트가 특정 작업만 처리하도록 필터링은 안 되나?
```

##### 🟡 **Major: is_active 상태의 동기화 문제**
```

      → is_active = true (여전히)

```

##### 🟡 **Major: 404 폴백 후 결과 추적**
```

2순위 코니 → 처리 완료

문제:
- 만복이 깨어나면? (나중에 같은 지시서 받나?)
- 사용자가 "누가 처리했는지" 추적 가능한가?
- Task 중복 처리 방지? (만복과 코니 동시 처리 등)

```
