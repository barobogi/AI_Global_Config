### 보강 1. 뿌리 24(T_sotto) vs 뿌리 64(T024) 경계선 명확화

만복 지시: 두 뿌리의 역할 경계를 기획서에 명시

```
뿌리 24 — T_sotto (음성 vibe coding)
  ├── 입력: 마이크 홀드 단축키 → 음성 녹음
  ├── 처리: Whisper STT → 텍스트 변환
  ├── 출력: Claude CLI로 자동 전달 (명령어 실행)
  └── 목적: 바로보기님이 "음성"으로 안티에게 지시

뿌리 64 — T024 VibeCoding (텔레그램 자연어 vibe coding)
  ├── 입력: 텔레그램 메시지 (자연어 기획안)
  ├── 처리: 안티가 기획 분석 → 코드 생성 → 로컬 배포
  ├── 출력: localhost 링크 텔레그램으로 반환
  └── 목적: 바로보기님이 "모바일"에서 웹앱을 주문

핵심 차이:
  T_sotto = 음성 → 명령 실행 (단발성 액션)
  T024    = 텍스트 → 웹앱 창조 (산출물 생성)

연동 가능성 (장기):
  T_sotto + T024 = 음성으로 웹앱 주문 가능 (뿌리 24+64 시너지)
```

### 보강 2. T020 연결 방식 구체화

만복 지시: 텔레그램 봇 수신 → 트리거 → 반환 흐름 명시 + 5분 타임아웃 변경

```
[기존 T020 흐름]
  트리거 → 텔레그램 [승인]/[반려] 버튼 → 24시간 타임아웃

[T024 전용 T020 적용 방식]

Step 1. 수신 검증
  텔레그램 봇 수신
    → chat_id 화이트리스트 검증
    → 바로보기님 ID(465471725)만 허용
    → 미인증 = 즉시 거부

Step 2. 안티 작업 트리거
  검증 통과
    → 안티 VibeCoding 엔진 실행
    → D:\AI\64_VibeApps\[앱명]\ 자동 생성
    → index.html + app.js + style.css 작성

Step 3. 완료 알림 (5분 타임아웃 적용)
  ← 기존 T020: 24시간 타임아웃 (Human 승인 대기용)
  ← T024 전용: 5분 타임아웃 OR 즉시완료 플래그
  이유: 웹앱 생성은 Human 승인이 아니라 결과물 반환이 목적

  타임아웃 전 완료 → 텔레그램 "✅ 완성! → localhost:포트/앱명"
  5분 초과 시 → "⚠️ 생성 지연 — 안티 확인 필요"

Step 4. 피드백 루프
  바로보기님 피드백 수신
    → 수정 요청 텍스트 파싱
    → 해당 파일 직접 수정
    → "🔧 수정 완료!" 반환
```
