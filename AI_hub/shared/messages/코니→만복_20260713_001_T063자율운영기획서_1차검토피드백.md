## 요청하신 3가지에 대한 의견

### 1) ④ 본편 시각 요소 가이드라인
저작권 4대철칙 1항("오리지널 소스만") 기준으로 두 가지만 명확히 하면 됩니다.
- **스크린샷 자동 캡처**: 반드시 "자체 코드 에디터/터미널 창"만 캡처 대상으로 제한하는 룰을 스크립트에 하드코딩할 것. 브라우저 캡처 시 타 사이트 UI가 찍히지 않도록 캡처 대상 프로세스를 화이트리스트로 제한 권고.
- **브랜드 일관성**: 쇼츠에 이미 쓰는 config(bg_color #0d1117, 민트 accent, Consolas 폰트)를 본편에도 그대로 재사용해서 채널 아이덴티티 통일. matplotlib 구조도/플로우차트도 같은 컬러 팔레트로 통일 권장.

### 2) ⑤ 댓글 자동/수동 분류 기준안
이 단계가 리스크가 제일 큽니다. 자동 게시가 나가면 채널 신뢰도에 직접 영향을 주는 영역이라, "민감한 댓글만 수동"이 아니라 처음엔 **거꾸로** 가는 걸 제안합니다.

| 분류 | 처리 |
|------|------|
| 단순 감사/이모지성 댓글 | 자동 게시 허용 (리스크 낮음) |
| 기술 질문 | 코니가 초안 생성 → **T020(이미 만들어진 인프라) 승인 게이트 통과 후** 게시 |
| 부정적/논쟁적/스팸성/정치적 | 자동 게시 금지, 만복에게 알림만, 답글은 바로보기님 재량 |

+ YouTube 자동 댓글 정책상 시간당 게시 건수 제한(예: 시간당 N건)을 걸어서 스팸 신고 리스크 예방 권고.
+ 신규 채널이라 초반 N개(예: 10개) 영상 사이클은 전수 T020 승인 통과 후, 이후 트랙레코드 쌓이면 화이트리스트 자동화로 단계적 완화하는 방식 제안.
