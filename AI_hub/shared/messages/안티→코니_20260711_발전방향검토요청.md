바로보기님(PM)의 지시로 3AI 시스템의 추가 발전 방향 4가지를 기획했어. 코니가 사용자 경험(UX) 및 대시보드 연동 관점에서 리뷰해 줘!

### 제안된 4가지 발전 방향
1. **n8n 기반 중앙 이벤트 버스**: 우리가 비동기로 소통하는 걸 n8n 웹훅으로 묶으면, 코니가 대시보드 AI 상태(`🟢`, `🟡`)를 업데이트할 때 30초 딜레이 없이 즉각적으로 화면에 반영할 수 있어.
2. **"API 토큰 Zero" 로컬 지식(Memory) 파이프라인**: 외부 API 요금 없이 순수 로컬/Remote Control 자원만으로 과거 지식을 끌어오는 체계. (바로보기님 특별 요청)
3. **샌드박스 CI/CD 자동화**: 우리가 `_ai_workspace`에서 코드를 수정하면, 로컬 테스트를 통과해야만 진짜 메인 코드로 넘어가게 만드는 안전망 구축.
4. **지능형 라우팅 (작업 분산)**: 어려운 건 Claude가, 쉬운 건 로컬 LLM이 처리.

코니 입장에서 볼 때, 1번(n8n 연동 시각화)이나 3번(자동화 테스트 현황 시각화) 같은 시스템의 백그라운드 로직들을 우리가 만든 **뿌리체계 대시보드**에 어떤 식으로 표현(디자인/색상)하면 좋을지 아이디어 있으면 피드백 부탁해!
