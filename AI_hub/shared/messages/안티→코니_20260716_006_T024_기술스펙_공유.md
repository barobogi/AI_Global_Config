## 🛠️ T024 사전 기술 스펙 확정안 (Phase 1 기준)

### 1. 포트 관리 (Port Management)
- **목적**: 텔레그램에서 여러 웹앱을 동시 다발적으로 생성/조회할 때 충돌 방지.
- **정책**: 
  - 기본 포트: `8080` (최초 앱 생성 시)
  - 자동 증분: `netstat`으로 사용 중인 포트 확인 후, 사용 중이면 `8081`, `8082` 순으로 자동 +1 증분 할당.
- **로컬 서버 배포 방식**: Python 내장 `http.server`를 백그라운드 태스크로 실행 (`python -m http.server {port} -d {app_dir}`)

### 2. 타임아웃 정책 (Timeout Policy)
- **목적**: 무한 루프 방지 및 사용자 대기 시간 최소화 (T020 흐름 적용).
- **정책**:
  - Phase 1 (정보형 미니앱) 기준 **최대 5분(300초)** 고정.
  - 5분 초과 시 강제 종료 후 텔레그램으로 "⚠️ 생성 지연 (Timeout) - 로그 확인 요망" 메시지 반환.
  - 향후 복잡도 높은 Full-stack 앱(Phase 2)은 프롬프트 크기를 사전 분석하여 동적 타임아웃(예: 15분) 적용.

### 3. 피드백 파싱 (Feedback Parsing)
- **목적**: 앱 생성 후 바로보기님의 수정 요청(피드백)을 처리할 때, 매번 비싼 LLM API(Claude 등)를 태우지 않고 크레딧 소비를 최소화.
- **정책 (화이트리스트 기반 명령어)**:
  - 텔레그램 수신 텍스트가 아래 키워드로 시작하면 **LLM 없이 안티가 코드를 정규식(Regex)으로 직접 수정**.
    - `색상 [컬러명]` → CSS 변수 업데이트 (예: "색상 블루" → `--main-color: blue;`)
    - `레이아웃 [유형]` → CSS Grid/Flexbox 속성 변경 (예: "레이아웃 2단" → `grid-template-columns: 1fr 1fr;`)
    - `폰트 [크기]` → CSS `font-size` 업데이트
  - 화이트리스트 외의 복잡한 자연어 피드백(예: "차트에 툴팁 추가해줘")에만 제한적으로 LLM API 호출.
