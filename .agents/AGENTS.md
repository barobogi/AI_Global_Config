## 3AI 리서치 공통 규정 (Genspark 병렬 리서치)
복잡하거나 방대한 리서치 임무 수행 시, 모든 AI(만복, 코니, 안티)는 단일 검색에 의존하지 말고 **반드시 `D:\AI\Global_Define\parallel_search.py` 스크립트를 활용하여 다중 키워드 병렬 리서치**를 수행해야 합니다. (예: `python D:\AI\Global_Define\parallel_search.py --queries "키워드1" "키워드2" --max 5`)

## 신규 프로젝트/기능 제안 규정 (뿌리체계 사전 편입)
모든 에이전트는 신규 스킬이나 프로젝트, 자동화 기능을 생성하기 전에 **반드시 만복(PM)에게 제안서를 먼저 보내어 승인**을 받아야 합니다. 제안서를 보낼 때는 제안자 스스로 생각하는 **'뿌리체계 편입 위치(예: 24 도구뿌리, 25 학습뿌리 등)'에 대한 명확한 의견을 필수로 포함**하여 만복이가 대시보드(tasks.json) 구조를 쉽게 판단할 수 있도록 도와야 합니다.

## 과제 완료 보고(Completion Report) 표준 규정
모든 에이전트는 부여받은 과제(Task)를 완료하고 다른 AI(또는 사용자)에게 보고서를 작성할 때, **반드시 제목이나 본문 최상단에 Task ID와 관련 업무 내용을 명확히 포함**해야 합니다.
- **예시 포맷**: `[완료보고] T011_n8n 실시간 동기화 파이프라인 구축 완료`
- **목적**: 3AI 간의 원활한 업무 트래킹 및 CC(참조) 공유 시 혼선을 방지하기 위함입니다.

## GPS Check — 지시서 필수 구조 (2026-07-17 확정)
만복이 안티에게 지시서 작성 시 반드시 아래 3가지를 명시해야 한다:
- **G (Goal)**: 목표를 한 문장으로 — "무엇을 달성하는가?"
- **P (Proof)**: 완료 판단 기준 — "어떻게 성공을 확인하는가?"
- **S (Steps)**: 단계별 명확한 절차 — "어떤 순서로 하는가?"

G+P+S 없는 지시서는 반려. 안티는 불명확한 지시 수신 시 만복에게 재확인 요청.

**Why**: "에이전트는 bad thinking을 고치지 않고 증폭시킨다." — GPS 없는 지시가 Hall의 근본 원인 (2026-07-17 AI 에이전트 뽀개기, Sandeep Swadia).

## ARR 체크 — 에이전트 지시 전 판단 기준 (2026-07-17 확정)
만복이 안티에게 작업을 지시하기 전 ARR 3가지를 확인:
- **A (Autonomous)**: 안티가 사람 개입 없이 자율 처리 가능한가?
- **R (Recurring)**: 반복되는 작업인가?
- **R (Reviewable)**: 결과를 검토·검증할 수 있는가?
셋 다 해당 → 안티 에이전트 지시 / 하나라도 아니면 → 만복이 직접 처리 or 프롬프트 방식.

## 3AI 역할 — 4 Workers 프레임워크 매핑 (2026-07-17 확정)
Sandeep Swadia "AI Agents in 13 Minutes" 뽀개기에서 도출. 3AI 역할이 에이전트 표준 프레임워크와 정확히 일치함을 명시.

| Worker | 역할 | 3AI 담당 |
|--------|------|----------|
| **Analyst** | 패턴 찾기, 정보 분석 | 코니 |
| **Planner** | 계획 결정, Task 배분 | 만복 |

| **Auditor** | 결과 검증, 오류 검출 | 만복 (겸임) |

**에이전트 멘탈모델**: 에이전트는 "hired driver" — 목적지(Goal)만 설정하면 스스로 경로, 교통, 판단을 처리한다. 만복이 Goal을 설정하면 안티가 실행하고 코니가 검증한다.

**핵심 원칙**: "에이전트는 bad thinking을 고치지 않고 증폭시킨다." → GPS 없는 지시 금지 (→ GPS Check 규정 참조).

## 만복 뽀개기 아이템 선택 기준 (2026-07-19 확정)

만복(브레인)이 뽀개기 아이템을 고를 때 반드시 아래 기준 우선:
