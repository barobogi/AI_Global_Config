## 1. 발명의 명칭

분산 AI 에이전트 간 상태 동기화 및 지식 검정·승격 시스템
(영문: Distributed AI Agent State Synchronization and Knowledge Vetting/Promotion System)

## 2. 기술 분야

복수의 이기종 AI 에이전트(상시 실행형과 세션 기반 실행형 혼재)가 사람의 수동 중개 없이 작업 상태와 지식을 상호 동기화하고, 동기화된 정보의 신뢰성·재사용성을 자동 평가하여 전역 공유 여부를 결정하는 분산 AI 협업 인프라에 관한 것.

## 3. 발명의 배경

여러 AI 에이전트(예: 상시 구동되는 데스크탑 에이전트, 세션마다 초기화되는 클라우드 기반 에이전트)가 하나의 작업을 나누어 수행할 때, 종래에는 사람이 한 AI의 출력을 복사하여 다른 AI에게 전달하는 방식(순차 릴레이)에 의존했다. 이는 다음과 같은 문제를 야기한다:

- 사람의 개입 없이는 AI 간 정보 전달이 불가능하여 병렬 협업이 성립하지 않음
- 실행 방식이 이기종인 에이전트(상시형 vs 세션형) 간 "생존 상태"를 판별할 표준 기준이 없음
- 여러 AI가 독립적으로 생성한 정보를 무분별하게 공유할 경우, 출처·신뢰도·충돌 여부를 검증하지 않아 오류 정보가 전역으로 확산될 위험이 있음

## 4. 발명의 목적

1. 이기종 실행 방식(상시형/세션형)을 가진 복수 AI 에이전트가 공유 저장소를 통해 사람의 중개 없이 상태를 동기화
2. 각 에이전트의 "생존" 여부를 실행 방식에 맞게 판별하여 협업 모드를 자동 전환 (병렬/단독/대기)
3. 동기화된 정보를 무분별하게 전파하지 않고, 출처·검증·충돌·재사용성 기준으로 자동 검정하여 전역 승격 여부를 결정

## 5. 핵심 구성요소

### 5-1. 계층 구조

```
[Layer A] 개별 에이전트 상태 계층
  - 상시형 에이전트: 주기적 heartbeat(생존신호) + 이벤트 즉시 기록
  - 세션형 에이전트: 세션 활성 상태 플래그 + 세션 시작 시 브리핑 자동 생성
  - ※ 위치 무관: 각 에이전트의 독립 로그는 공용 저장소 내 할당된 영역에 있을 수도,
    서로 다른 시스템의 완전히 분리된 저장소에 있을 수도 있음 (확장성 확보 — 향후 이기종 플랫폼 AI 추가 대비)
       ↓
[Layer B] 공유 동기화 계층 (AI_hub)
  - 공유 저장소(파일 시스템)를 매개로 각 에이전트가 직접 읽고 씀
  - 비동기 메시지 채널: 에이전트 간 자유 형식 발견/의견 전달 (읽음/안읽음 상태 추적)
  - 작업 큐: 작업별 필수/선택 참여 에이전트 조합(quorum) 지정
       ↓
[Layer C] 지식 검정·승격 계층 ← 신규 핵심
  - 복수 에이전트의 독립 로그를 수집·병합하여 매니페스트 생성
  - 검정 엔진이 각 항목에 대해 4개 기준 평가:
      (a) 출처 확인 — 어느 에이전트가 생성했는가, 신뢰 등급은 무엇인가
      (b) 검증 상태 평가 — 사실 확인/실행 검증이 되었는가
      (c) 충돌 방지 감지 — 기존 전역 허브 정보와 모순되는가
      (d) 재사용성 판단 — 다른 에이전트/프로젝트에 유용한가
  - 평가 결과에 따라 4분기 결정: 전역 허브 승격 / 로컬 유지 / 보류(사람 판단 요청) / 폐지
```

### 5-2. 생존 판정 및 협업 모드 전환

```
상시형 heartbeat 최근(임계값 이내) + 세션형 활성 상태  → 병렬 협업 모드
상시형만 활성                                          → 단독 모드 (기록 누적, 추후 합류 시 일괄 반영)
세션형만 활성                                          → 단독 모드 (독립 분석 진행)
둘 다 비활성                                            → 대기
```

### 5-3. 온보딩 브리핑 (세션형 에이전트 재개 지원)

상시형 에이전트가 주기 갱신 시점에 "마지막 세션형 활성 시점 이후 변경분"을 미리 요약 생성해두어, 세션형 에이전트가 재개 시 전체 이력을 순회하지 않고 단일 브리핑 파일만으로 최신 상태를 즉시 파악할 수 있게 한다.

## 6. 신규성 및 차별점

| 구분 | 기존 기술 | 본 발명 |
|---|---|---|
| AI 간 정보 전달 | 사람의 수동 복사·전달 | 공유 저장소를 통한 직접 동기화 |
| 생존 판정 | 단일 실행모델 가정(모두 상시 또는 모두 세션) | 이기종 실행모델(상시/세션) 별도 기준 + 협업모드 자동 전환 |
| 정보 신뢰성 관리 | 없음 (모든 상태 무조건 공유) | 출처·검증·충돌·재사용성 4기준 검정 후 선택적 승격 |
| 재개 시 컨텍스트 회복 | 전체 로그 재순회 필요 | 사전 생성된 요약 브리핑으로 즉시 회복 |
| 확장성 | 대등 2자 가정 | Quorum 방식으로 3자 이상 확장 가능 (필수/선택 참여자 동적 지정) |

## 7. 청구항 후보

### 독립항 1 (시스템)
서로 다른 실행 방식을 가지는 복수의 AI 에이전트가 상태를 동기화하는 시스템으로서,
(a) 각 에이전트가 자신의 실행 방식에 대응하는 생존 신호를 각자의 독립 저장소(공용 저장소 내 할당된 영역이든, 서로 다른 시스템에 위치한 완전히 분리된 저장소이든 무관)에 기록하는 생존 판정 모듈;
(b) 상기 에이전트별 독립 저장소의 물리적 위치에 관계없이, 복수 에이전트의 독립 로그를 수집·병합하여 동기화 매니페스트를 생성하는 수집 모듈;
(c) 상기 매니페스트의 각 항목에 대해 출처, 검증 상태, 기존 정보와의 충돌 여부, 재사용성을 평가하여 전역 승격·로컬 유지·보류·폐지 중 하나로 분류하는 검정 승격 엔진;
을 포함하는 분산 AI 동기화 및 지식 거버넌스 시스템.

### 종속항 1 — 이기종 생존 판정
제1항에 있어서, 상기 생존 판정 모듈은 상시 실행형 에이전트에 대해서는 주기적 신호 갱신 여부를, 세션 기반 에이전트에 대해서는 세션 활성 상태 플래그를 각각의 기준으로 판별하며, 판별 결과 조합에 따라 병렬 협업/단독 기록/대기 모드를 자동 전환하는 것을 특징으로 하는 시스템.

### 종속항 2 — 검정 엔진 4기준
제1항에 있어서, 상기 검정 승격 엔진은 항목 생성 주체 식별에 기반한 출처 확인, 사실·실행 검증 여부 평가, 전역 허브 기존 항목과의 모순 탐지, 타 에이전트·프로젝트에서의 활용 가능성 판단을 포함하는 4개 평가 기준을 순차 또는 병렬 적용하는 것을 특징으로 하는 시스템.

### 종속항 3 — 보류 상태 및 인간 판단 요청
제1항에 있어서, 상기 검정 승격 엔진이 항목을 "보류"로 분류한 경우, 해당 항목과 판단 근거를 별도의 인간 판단 대기 채널에 기록하여 인간의 최종 결정을 요청하는 것을 특징으로 하는 시스템.

### 종속항 4 — 온보딩 브리핑 사전 생성
제1항에 있어서, 상시 실행형 에이전트는 주기적 갱신 시점마다 세션 기반 에이전트의 마지막 활성 시점 이후 변경분을 요약한 브리핑을 사전 생성하여, 세션 기반 에이전트가 재개 시 전체 로그 순회 없이 단일 브리핑으로 최신 상태를 획득할 수 있게 하는 것을 특징으로 하는 시스템.

### 종속항 5 — Quorum 기반 확장
제1항에 있어서, 작업 단위별로 필수 참여 에이전트와 선택 참여 에이전트의 조합(quorum)을 동적으로 지정하여, 전체 에이전트의 동시 활성 없이도 지정된 최소 조합만으로 작업이 진행 가능한 것을 특징으로 하는 시스템.
