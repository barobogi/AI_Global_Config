## 📋 Abstract (발명 요약)

**Title**: Automated Determination System for Commercial Use Rights of AI-Generated Content

**Summary**:
This invention relates to an automated system that determines whether AI-generated content (video, image, audio) from generative platforms (Kling, Sideo, Runway, etc.) can be commercially utilized across different publishing platforms (YouTube, TikTok, etc.) based on real-time verification of three-party terms: (1) the generative platform's commercial licensing policy, (2) the publishing platform's monetization policy, and (3) the user's current subscription plan. 

Unlike existing content verification systems that focus on content authenticity or metadata, this invention provides **rights chain tracking** - a continuous verification from content generation time through publication time, automatically determining: "Upload permitted" / "Upload prohibited" / "Plan upgrade required".

**Key Innovation**: Real-time cross-platform terms verification with automated rights chain tracking.

## 2. 발명의 기술 분야

본 발명은 생성 AI(Generative AI) 기술 분야에서, 특히 AI 기반 비디오/이미지 생성 서비스(Kling AI, Sideo, Runway, DALL-E 등)의 약관과 콘텐츠 업로드 플랫폼(YouTube, TikTok, Instagram 등)의 정책을 실시간으로 검증하여, 사용자가 생성한 콘텐츠의 상업화 가능 여부를 자동으로 판단하는 기술에 관한 것이다.

## 3. 종래 기술의 문제점

### 3-1. 현 상황의 문제

**AI 콘텐츠 상업화의 혼란 상황**:

```
사용자의 고민:
"Kling AI로 만든 영상을 유튜브에 올려도 수익화해도 되나?"

현재 해결 방법:
1. Kling 약관 읽기 (5-10분)
2. 유튜브 정책 읽기 (5-10분)
3. 자신의 플랜 확인 (1-2분)
4. 수동으로 판단 (주관적, 오류 가능)

문제점:
❌ 시간 소요 (총 15-25분)
❌ 오류 가능성 높음
❌ 플랜 변경 시 재확인 필수
❌ 약관 업데이트 놓칠 수 있음
```

### 3-2. 기존 기술의 한계

**콘텐츠 검증 시스템 (유사 특허들)**:
- 콘텐츠 등록 및 관리 중심 (US 12417393)
- AI 에이전트 검증 중심 (US20260017525A1)
- 메타데이터 검증 중심
- **상업 이용 권리 판단 불가능**

**약관 분석 자동화**:
- 단일 플랫폼 분석만 가능
- **3자 연계 검증 불가능**
- 정책 변경 시 수동 업데이트

---

## 4. 발명의 목적

본 발명의 목적은 AI 생성 콘텐츠의 상업화 권리를 **실시간으로 자동 판단**하여, 사용자가 생성 시점부터 업로드 시점까지 **안심하고 콘텐츠를 활용**할 수 있도록 하는 것이다.

구체적으로:
1. **3자 약관 자동 크로스 검증** (생성 플랫폼 + 업로드 플랫폼 + 플랜)
2. **권리 체인 추적** (생성 → 업로드 → 수익화까지 연속 검증)
3. **자동 판단 및 가이드** ("가능" / "불가" / "업그레이드 필요")
4. **정책 변경 자동 감지** (약관 업데이트 즉시 반영)

## 5. 발명의 핵심 구성

### 5-1. 3자 연계 검증 아키텍처

```
┌──────────────────────────────────────────────┐
│  사용자: 콘텐츠 생성                         │
│  (Kling AI, Sideo, Runway)                  │
└────────────────────┬───────────────────────┘
                     ↓
        ┌────────────────────────┐
        │   자동 판단 시스템    │
        │   (3자 검증 로직)     │
        └────────┬───────────────┘
         ↙        ↓         ↘
    ┌─────────┐  │  ┌─────────┐
    │ 생성    │  │  │ 업로드  │
    │ 플랫폼  │  │  │ 플랫폼  │
    │ 약관    │  │  │ 정책    │
    │ (DB)   │  │  │ (DB)   │
    └─────────┘  │  └─────────┘
                 │
            ┌────┴─────┐
            │ 사용자   │
            │ 플랜     │
            │ (DB)    │
            └──────────┘

결과: "가능" / "불가" / "업그레이드"
```

### 5-2. 핵심 구성요소

#### **[구성요소 1] 약관 크롤링 + 파싱 엔진**
- 각 생성 플랫폼(Kling, Sideo, Runway)의 약관을 주기적으로 추출
- NLP 기반 권리 정보 자동 추출 ("상업 이용 허가", "수익 배분" 등)
- 정책 변경 감지 및 자동 업데이트

#### **[구성요소 2] 업로드 플랫폼 정책 DB**
- 유튜브, TikTok, Instagram 등의 수익화 정책 저장
- 플랫폼별 제한사항 (AI 생성 콘텐츠 관련)
- 수익 배분 모델 정보

#### **[구성요소 3] 사용자 플랜 관리**
- 각 생성 플랫폼의 구독 플랜 정보
- 플랜별 상업 이용 권한 매핑
- 실시간 플랜 상태 추적

#### **[구성요소 4] 자동 판단 엔진**
- 규칙 기반 로직: IF (생성 플랫폼 허용) AND (업로드 플랫폼 허용) AND (플랜 포함) → "가능"
- 복합 시나리오 처리: 부분 허용, 수익화만 제한 등
- 이유 제시: "왜 불가능한가?"를 사용자에게 명확히 설명

## 8. 비자명성 (Non-Obviousness)

### 8-1. 당업자의 도달 불가능성

**당업자 정의**: AI 플랫폼 개발자, 콘텐츠 관리 시스템 엔지니어

**당업자가 도달할 수 없는 이유**:

```
1. 단일 플랫폼 사고 (관습적)
   → 각 플랫폼은 자신의 정책만 관리
   → 타 플랫폼과의 크로스 검증 관습 없음

2. 정책 변경의 복잡성
   → 수십 개 플랫폼의 약관 실시간 추적 어려움
   → 수동 업데이트 비용 높음

3. 권리 체인의 개념 부재
   → 기존 기술: "콘텐츠 검증" (미디어 포커스)
   → 우리 발명: "권리 검증" (법리/정책 포커스)
   → 이 두 분야의 통합이 자명하지 않음
```

### 8-2. 결론

**비자명성 강도: 높음** ✅

당업자가 자동화된 콘텐츠 관리 시스템을 개발하더라도, 
**3자 약관의 실시간 크로스 검증**이라는 개념과 
**권리 체인 추적**이라는 설계는 
기존 기술 조합으로는 자명하게 도달할 수 없다.
