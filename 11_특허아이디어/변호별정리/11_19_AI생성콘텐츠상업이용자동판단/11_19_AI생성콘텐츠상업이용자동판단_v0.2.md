## 📋 Abstract (발명 요약)

**Title**: Automated Determination System for Commercial Use Rights of AI-Generated Content

**Summary**:
This invention relates to an automated system that determines whether AI-generated content (video, image, audio) from generative platforms (Kling, Sideo, Runway, etc.) can be commercially utilized across different publishing platforms (YouTube, TikTok, etc.) based on real-time verification of three-party terms: (1) the generative platform's commercial licensing policy, (2) the publishing platform's monetization policy, and (3) the user's current subscription plan. 

Unlike existing content verification systems that focus on content authenticity or metadata, this invention provides **rights chain tracking** - a continuous verification from content generation time through publication time, automatically determining: "Upload permitted" / "Upload prohibited" / "Plan upgrade required".

**Key Innovation**: Real-time cross-platform terms verification with automated rights chain tracking.

### 3-1. 실제 시나리오: 바로보기님과 만복 PM의 직접 경험

#### **사건 1: BytePlus Sideo 크레딧 확인 불가**

```
상황:
- Sideo(BytePlus)로 비디오 생성 테스트 전
- 무료 크레딧 잔여 확인 필요

수동 과정:
1. BytePlus Console 접속
2. Billing/Credits 메뉴 탐색
3. 잔여 크레딧 확인 시도

실패 원인:
- UI가 복잡하고 비직관적
- 크레딧 정보 위치 불명확
- 기술적 기술이 필요 (일반 사용자 입장에선 어려움)

결과:
❌ 잔여 크레딧 확인 불가
❌ 상업 이용 가능 여부 판단 불가
→ Kling AI로 대체 (차선책)
```

**이것이 증명하는 것**:
- 사용자가 플랫폼 정책을 쉽게 확인할 수 없음
- 기술적 장벽 존재
- 자동 판단 시스템의 필요성

수동 과정:

결과:
❌ 약관 내용 확인 불가

→ 바로보기님에게 "직접 확인해야 한다"고 안내

**이것이 증명하는 것**:
- 사람이 직접 약관을 읽어야 함

- 자동 판단을 위해서는 플랫폼과의 특별한 협약이 필요

## 4. 청구항 (Claims)

### **청구항 1 (독립 청구항)**

```
AI 생성 콘텐츠의 상업 이용 가능 여부를 판단하는 시스템으로서,

(b) 콘텐츠 업로드 플랫폼(YouTube, TikTok 등)의 정책에서
    AI 생성 콘텐츠 관련 수익화 정책을 실시간으로 동기화하는 
    플랫폼 정책 관리 모듈;

(d) (a)~(c)의 정보를 종합하여 규칙 기반 로직으로
    "업로드 가능" / "업로드 불가" / "플랜 업그레이드 필요"를
    자동 판정하는 판단 엔진;

(e) 판단 결과를 사용자에게 제시하고, 약관 변경 시
    자동으로 재검증하는 권리 체인 추적 모듈

을 포함하는 것을 특징으로 하는 
AI 생성 콘텐츠 상업 이용 자동 판단 시스템.
```

```

- 추출된 정보를 구조화된 데이터(JSON)로 변환하여 저장하며,

- 매일 정기적으로 약관 업데이트를 감지하여 자동 갱신하는

것을 특징으로 하는 청구항 1의 AI 생성 콘텐츠 상업 이용 자동 판단 시스템.
```

### **청구항 3 (종속 청구항 - 3자 크로스 검증)**

청구항 1에 있어서, 상기 판단 엔진은:

```
다음의 논리식으로 판정을 수행한다:

IF (생성_플랫폼_허가 = TRUE) 
   AND (업로드_플랫폼_허가 = TRUE) 
   AND (사용자_플랜_포함 = TRUE)
THEN 판정 = "업로드 가능"

ELSE IF (플랜_업그레이드_시_허가 = TRUE)
THEN 판정 = "플랜 업그레이드 필요"
설명 = "현재 플랜에서는 불가, Pro 플랜으로 업그레이드 시 가능"

ELSE
THEN 판정 = "업로드 불가"
설명 = "해당 플랫폼에서 상업 이용 미지원"

을 특징으로 하는 청구항 1의 AI 생성 콘텐츠 상업 이용 자동 판단 시스템.
```

  추후 분쟁 시 증거 자료로 활용 가능하게 하는

것을 특징으로 하는 청구항 1의 AI 생성 콘텐츠 상업 이용 자동 판단 시스템.

```
AI 생성 콘텐츠의 상업 이용 가능 여부를 판단하는 방법으로서,

(1) 사용자로부터 생성 플랫폼명, 타겟 업로드 플랫폼,

(4) 사용자 플랜이 위 약관 및 정책에서 
    요구하는 권리를 포함하는지 검증하는 단계;

    최종 판정을 내리고 사용자에게 표시하는 단계;

(6) 향후 약관 변경을 감지하면 자동으로 재검증을 수행하고
    필요시 사용자에게 알림을 전송하는 단계

를 포함하는 것을 특징으로 하는
AI 생성 콘텐츠 상업 이용 자동 판단 방법.
```

## 5. 종래 기술의 문제점 (개선)

### 5-1. 실증된 문제

위의 실제 사례(사건 1~3)에서 나타난 문제:

```
1. 사용자 수동 확인의 어려움
   - 약관 접근 차단 (HTTP 446 등)
   - UI 복잡성
   - 기술적 장벽

2. 시간 소요
   - 약관 하나당 5-10분
   - 3-5개 플랫폼 확인 = 15-25분

3. 오류 가능성
   - 약관 해석의 모호성
   - 플랜별 권리 차이 파악 어려움
   - 20-30% 오류율

4. 정책 변경 추적 불가
   - 플랫폼이 약관을 변경해도 사용자는 모름
   - 기존에 허락된 콘텐츠가 갑자기 업로드 불가 상태로 변경 가능
```

### 5-2. 기존 기술의 한계

```
기존 콘텐츠 검증 시스템:
- 콘텐츠의 진위 여부만 확인 (authenticity)
- 메타데이터 검증 (metadata)
- 상업 이용 권리는 검증하지 않음 ❌

기존 약관 분석 도구:
- 단일 플랫폼만 분석 가능
- 3자 크로스 검증 불가능 ❌
- 수동 업데이트 필요
- 실시간 추적 불가능 ❌
```
