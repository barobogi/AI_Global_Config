## 2. 🎯 특허 11_18 진보성 핵심 논리 (LangGraph 한계 기반)

### 2.1 LangGraph의 본질적 제약: "단일 Python 런타임 내 그래프"
LangGraph의 모든 멀티에이전트 구조는 영상에서 명확하게:
- **하나의 그래프(단일 런타임)** 안에서 동작하거나
- **서브그래프(중첩 구조)**로 확장되더라도 동일한 Python 실행 환경 안에 포함됨

이는 **스테이트(공유 메모리)를 같은 프로세스 내에서만 공유**한다는 근본적인 제약을 의미함.

### 2.2 LangGraph로 해결 불가능한 문제 (본 발명의 존재 근거)

| 제약 | LangGraph | 본 발명 (11_18) |
|------|-----------|----------------|
| **에이전트 구동 환경** | 단일 Python 런타임 (하나의 프로세스) | 이질적 멀티 플랫폼 (CLI + GUI 샌드박스 앱) |
| **스테이트 공유 방식** | 같은 메모리 내 공유 객체 | 로컬 파일시스템 + DB (Shared Brain) |
| **상용 앱과의 연동** | 불가능 (Desktop Claude 등) | MCP 브릿지로 실현 |
| **샌드박스 대응** | 고려 없음 (동일 환경 전제) | 핵심 설계 요소 |
| **플랫폼 간 실시간 동기화** | 불가능 | ✅ 달성 |

### 2.3 비자명성(Non-obviousness) 확립 논리
LangGraph가 9시간짜리 풀강의를 통해 가르치는 **최첨단 멀티에이전트 기술**조차도:
1. 상용 데스크탑 앱(Desktop Claude)과 백그라운드 CLI 에이전트를 **동기화하는 방법을 제시하지 않음**
2. 샌드박스 환경의 에이전트와 자율형 에이전트의 협업 구조를 **전혀 다루지 않음**
3. 이종 플랫폼 간 통신망의 개념 자체가 **LangGraph 아키텍처의 설계 범위 밖**임

→ **결론: LangGraph를 완벽히 숙지한 당업자라도 본 발명(MCP 브릿지 기반 이종 플랫폼 동기화)에 자명하게 도달할 수 없음. 비자명성 ✅ 확립.**
