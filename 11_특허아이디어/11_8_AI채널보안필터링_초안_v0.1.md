# 11_8 특허 초안 v0.1
## AI 채널 전송 전 실시간 보안 필터링 시스템

작성: 2026-06-30 | 상태: 브레인스토밍용 초안

---

## 1. 배경 및 문제 정의

사용자는 Claude, GPT, Cowork 등 다양한 AI 채널을 일상적으로 사용한다.
이 과정에서 주민번호, 계좌번호, 의료 정보, 사내 기밀 등 민감 정보를
**실수로** 외부 AI 서버에 전송하는 사고가 발생할 수 있다.

기존 DLP(Data Loss Prevention)는 기업 네트워크 레이어에서 동작하지만,
**개인이 멀티 AI 채널을 자유롭게 쓰는 환경**에는 적용되지 않는다.

---

## 2. 핵심 발명 아이디어

### 기본 개념
> 사용자가 AI에 입력하는 내용을 **로컬에서 실시간 분석**하여,
> 민감 정보가 감지되면 **외부 서버 전송 전에** 차단하거나 보안 채널로 리다이렉트하는 시스템

### 작동 흐름
```
사용자 타이핑 중
    ↓ (로컬 실시간 분석 — 서버 전송 없음)
민감 정보 패턴 감지?
    ├── No  → 그냥 전송
    └── Yes → [옵션 A] 경고 팝업 + 사용자 확인
               [옵션 B] 자동으로 보안 채널 전환
               [옵션 C] 해당 부분 마스킹 후 전송
```

---

## 3. 차별화 포인트 (기존 DLP 대비)

| 구분 | 기존 DLP | 본 발명 |
|------|----------|---------|
| 적용 대상 | 기업 네트워크 | 개인 멀티 AI 채널 |
| 처리 위치 | 네트워크 서버 | 로컬 (클라이언트) |
| 동작 시점 | 전송 후 감지 가능 | **타이핑 중 실시간** |
| 채널 전환 | 없음 | AI 채널 자동 리다이렉트 |
| 개인화 | 없음 | 사용자별 민감도 설정 |

---

## 4. 청구항 후보

### 독립 청구항
- 사용자 입력을 로컬에서 실시간 분석하여 민감 정보를 AI 서버 전송 전에 차단하는 방법

### 종속 청구항
- 민감도에 따라 채널을 자동 전환하는 기능
- 사용자가 민감 정보 기준을 커스텀 설정하는 기능  
- 스마트폰 환경에서 외부 네트워크 접속 시 보안 레벨 자동 상향

---

## 5. 구현 가능성

- 로컬 소형 LLM or 패턴 매칭으로 구현 가능 (외부 API 불필요)
- 브라우저 익스텐션 or 앱 레이어로 구현 가능
- 스마트폰 키보드 레이어에 탑재 가능 (HW 확장 가능성)
