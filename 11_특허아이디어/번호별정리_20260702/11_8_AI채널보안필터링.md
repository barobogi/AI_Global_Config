# 11_8 — AI 채널 전송 전 실시간 보안 필터링 시스템

**상태**: 유지 (흡수 권고, 실제 문서 반영 미완료) — 특허2 보안청구항과 겹침
**작성일**: 2026-06-30 | **원본**: `11_8_AI채널보안필터링_초안_v0.1.md`

---

## 배경 및 문제 정의
사용자는 Claude, GPT, Cowork 등 다양한 AI 채널을 일상적으로 사용. 이 과정에서 주민번호, 계좌번호, 의료 정보, 사내 기밀 등 민감 정보를 실수로 외부 AI 서버에 전송하는 사고 발생 가능. 기존 DLP(Data Loss Prevention)는 기업 네트워크 레이어에서 동작하지만, 개인이 멀티 AI 채널을 자유롭게 쓰는 환경에는 적용되지 않음.

## 핵심 발명 아이디어
사용자가 AI에 입력하는 내용을 로컬에서 실시간 분석하여, 민감 정보가 감지되면 외부 서버 전송 전에 차단하거나 보안 채널로 리다이렉트하는 시스템.

```
사용자 타이핑 중
    ↓ (로컬 실시간 분석 — 서버 전송 없음)
민감 정보 패턴 감지?
    ├── No  → 그냥 전송
    └── Yes → [A] 경고 팝업+사용자 확인 / [B] 자동 보안채널 전환 / [C] 해당 부분 마스킹 후 전송
```

## 차별화 포인트 (기존 DLP 대비)

| 구분 | 기존 DLP | 본 발명 |
|---|---|---|
| 적용 대상 | 기업 네트워크 | 개인 멀티 AI 채널 |
| 처리 위치 | 네트워크 서버 | 로컬(클라이언트) |
| 동작 시점 | 전송 후 감지 가능 | 타이핑 중 실시간 |
| 채널 전환 | 없음 | AI 채널 자동 리다이렉트 |
| 개인화 | 없음 | 사용자별 민감도 설정 |

## 청구항 후보
- **독립**: 사용자 입력을 로컬에서 실시간 분석하여 민감 정보를 AI 서버 전송 전에 차단하는 방법
- **종속**: 민감도에 따른 채널 자동 전환 / 사용자 커스텀 민감정보 기준 설정 / 스마트폰 외부망 접속 시 보안레벨 자동 상향

## 구현 가능성
로컬 소형 LLM or 패턴 매칭으로 구현 가능(외부 API 불필요) · 브라우저 익스텐션 or 앱 레이어 · 스마트폰 키보드 레이어 탑재 가능(HW 확장성)

## 특허2와의 관계 및 남은 작업
특허2(멀티채널 오케스트레이션)의 보안청구항과 내용이 겹쳐 흡수 권고 상태이나, **실제 문서상 흡수 작업은 아직 반영되지 않음** — 다음 세션에서 특허2 명세서에 종속항으로 편입할지, 11_12(물리보안경계)와 통합할지 결정 필요.
