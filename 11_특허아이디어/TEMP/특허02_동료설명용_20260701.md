## 쉬운 비유

**원격 데스크탑**:
> 친구 집 컴퓨터를 내 화면에 띄워서 내가 직접 마우스 클릭하는 것.  
> 친구 집 전기 나가면? → 끝.

**본 발명**:
> 친구 집 컴퓨터가 꺼져도, 우리 둘이 **같은 메모장**을 공유하고 있어서  
> 내 컴퓨터에서 다른 AI가 그 메모장 읽고 이어서 일함.  
> "뇌"가 어딘가에 따로 살아있는 것.

---

## 기술적으로 정확하게

```
원격 데스크탑:
  [PC] = 실행 주체 + 컨텍스트(기억) 한 몸
  → PC 꺼지면 둘 다 소멸

본 발명:
  [외부 저장소] ← 컨텍스트(기억) 항상 여기 있음
       ↕
  [만복1] [만복2] [코니] [두복이] ... → 각자 저장소에 접속
  → 하나가 꺼져도 다른 AI가 동일 기억으로 바로 이어받음
```

---

## 왜 신뢰루트·권한검증이 필요한가

원격 데스크탑은 접속하면 전부 다 됨 (이분법).

본 발명은 AI가 **여럿**이라 채널마다 신뢰 수준이 다름:
- 데스크탑 CLI 접속 → 모든 권한
- 텔레그램 봇 → 알림/간단 조회만
- 모바일 앱 → 제한된 작업만

그래서 **권한 검증형 선택위임 + 최소 컨텍스트 라우팅 + 결과 검증 병합**이 구조적으로 필요.
