### 특허 가능성: 🔶 중간

**핵심 신규성 (3요소 조합)**
- watchdog 기반 세션 종료 자동 감지
- 이기종 AI 플랫폼 간 파일 기반 컨텍스트 전달
- AI 자신이 후계 AI를 위해 능동적으로 브릿지 문서 생성

**발견된 선행기술과 차이**
- MS/Google/OpenAI 특허: 모두 클라우드 종속 또는 단일 플랫폼
- 이기종 AI 간 파일 기반 자율 전달은 선행기술 미발견

**강화 청구항 제안**
> "파일 시스템 감시 모듈이 제1 AI 에이전트 세션 종료를 감지 → 제2 이기종 AI가 판독 가능한 구조화된 브릿지 파일 자동 생성 → 제2 AI가 세션 초기화 시 자동 로드하여 컨텍스트 복원하는 이기종 AI 에이전트 간 세션 컨텍스트 자동 인계 방법"

**권고**: SW 단독 특허 어려움 → **#7 멀티채널 오케스트레이션과 결합 출원** 시 청구 범위 강화

## 11_#3 — 외부 AI 위임 실행 패턴 (Multi-tier AI Delegation)

### 특허 가능성: 🔶 중간 (단독) / 🔷 중간~높음 (결합)

**핵심 신규성 (3요소 조합)**
- ① 비용 등급 기반 AI 역할 명시적 분리
- ② 로컬 파일 시스템 기반 플랫폼 독립 인터페이스
- ③ 판단 AI의 실행 AI 결과 2단계 검증 루프

**발견된 선행기술과 차이**
- AutoGen/LangGraph: 역할 분리 있으나 비용 등급 개념 없음
- AWS Bedrock: AWS 종속, 파일 인터페이스 없음
- MoE 라우팅: 단일 모델 내부, 외부 검증 루프 없음

**강화 청구항 제안**
> "제1 고비용 AI가 태스크 명세+품질기준 파일 생성 → 제2 저비용 AI가 반복 실행+결과 파일 저장 → 제1 AI가 품질기준 비교 후 합격/재실행/에스컬레이션 결정하는 플랫폼 독립적 파일 기반 다단계 AI 실행 위임 방법"

**종속항 추가**: 재실행 임계값 초과 시 자동 에스컬레이션, 실시간 비용 추적 모듈

**권고**: **호두AI(#1) 또는 멀티채널(#7)과 결합** → HW+SW 청구 가능, 특허 가능성 상승

## 11_#4 — 시장 결과 기반 투자 신호 자동 학습 시스템

### 특허 가능성: 🔶 중간~높음 ⭐ (한국 BM 특허 중 가장 높음)

**핵심 신규성 (3요소 조합)**
- ① 인간 판단 완전 배제 (자동 피드백 루프)
- ② 5일 수익률 단일 피드백 기준 (복잡한 보상함수 불필요)
- ③ 개인 투자자 수준 경량 파이프라인

**발견된 선행기술과 차이**
- 기존 RL 트레이딩: ①은 충족, 대규모 인프라 필요 (③ 불충족)
- 기존 백테스팅: 사후 검증, 실시간 자동 갱신 없음
- 기존 온라인 러닝: 복합 보상함수 사용 (② 불충족)
- **→ 3요소 조합의 국내 독립 특허 매우 희소 가능성**

**강화 청구항 제안**
> "신호 생성+타임스탬프 → N영업일 후 자동 결과가격 수집 → 인간 개입 없이 성과점수 산출 → 가중치 자동 갱신하는, 개인 투자자 단말기 실행 가능 경량 파이프라인 투자 신호 최적화 방법"

**종속항 추가**: N=3~10영업일 한정, EMA 방식 가중치 갱신

**권고**: **변리사 상담 2순위** (호두AI 다음). Improve_stock 코드를 실시예로 첨부 시 명세서 강화. 한국 단독 출원 유리.
