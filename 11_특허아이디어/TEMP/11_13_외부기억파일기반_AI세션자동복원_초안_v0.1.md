# 11_13 — 외부 기억 파일 기반 AI 에이전트 세션 자동 복원 시스템

**등록일**: 2026-07-02  
**작성**: 만복 (오늘 아침 MCP + Project Instructions 결합 실증에서 발굴)  
**상태**: 초안 v0.1 — 선행검색 전

---

## 핵심 아이디어

세션 간 기억이 없는 AI 에이전트가,
**외부 기억 파일(상태 요약 파일) + MCP filesystem + 행동 지침(Project Instructions)** 을 결합하여
새 세션 시작 시 이전 상태를 자동으로 복원하는 시스템.

---

## 발굴 배경 (오늘 아침)

```
[기존]
새 코니 세션 → 기억 없음 → 바로보기님이 수동 붙여넣기 → 동기화

[오늘 구현·실증]
새 코니 세션
  → Project Instructions: "세션 시작 시 코니_quick_sync.md 읽어라"
  → MCP filesystem: 파일 직접 읽기
  → 자동 복원 완료 (수동 개입 0)
```

---

## 기존 특허와의 차이

| | 특허3 (분산 AI 동기화) | 11_2 (세션 컨텍스트 동기화) | **11_13 (세션 자동 복원)** |
|---|---|---|---|
| 대상 | AI ↔ AI 간 상태 공유 | 세션 간 컨텍스트 전달 | 단일 AI의 자기 기억 복원 |
| 방식 | 공유 파일 허브 | 사람이 컨텍스트 전달 | MCP + 지침 결합 자동 실행 |
| 핵심 | 동기화 인프라 | 컨텍스트 전달 | **세션 시작 트리거 → 자동 복원** |
| 사람 개입 | 최소 | 있음 | **없음** |

---

## 구성 요소 3가지

1. **외부 기억 파일** (코니_quick_sync.md): 최신 상태 압축 요약
2. **MCP filesystem**: AI가 로컬 파일 직접 접근하는 통로
3. **Project Instructions 트리거**: "세션 시작 시 파일 읽어라" 자동 실행 지시

세 가지가 결합될 때만 완전 자동화 달성. 하나라도 빠지면 사람 개입 필요.

---

## 청구항 방향 (초안)

**독립항 1**: AI 에이전트가 새 세션 시작 시, 행동 지침에 설정된 트리거에 따라 외부 파일 접근 프로토콜(MCP)을 통해 기억 파일을 자동 로드하여 이전 세션 상태를 복원하는 방법

**종속항 후보**:
- 기억 파일을 별도 에이전트(master_watch)가 주기적으로 자동 갱신하는 방법
- 기억 파일을 핵심 요약(MCU)으로 압축하여 로드 시간 및 컨텍스트 소비 최소화하는 방법
- 복수의 이종 AI 플랫폼(Claude, GPT 등)이 동일한 외부 기억 파일을 공유하여 각자 복원하는 방법
- 세션 복원 성공/실패 여부를 사용자에게 1줄 요약으로 보고하는 방법
