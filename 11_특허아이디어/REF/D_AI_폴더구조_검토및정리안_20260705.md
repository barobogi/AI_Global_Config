## 4. 제안 — 위험도별로 3단계

### A. 지금 당장 해도 안전 (이동 없음, 등록부만 손질)
- `claude-trading-skills`를 `.cowork-projects-registry.json`에서 `enabled: false` 처리 (외부 레포 자동 push 방지)
- `n8n_data ` 폴더명 공백 제거 (참조하는 스크립트 1곳만 확인하면 안전)
- 루트에 `PROJECTS_INDEX.md` 하나 새로 작성 — 폴더 이동 없이 위 카테고리 표(활성/휴면/레거시/외부/인프라)만 문서화. 이동은 안 하지만 "이게 뭐였지" 헷갈림은 바로 해소됨

### B. 저위험 정리 (등록 안 된 폴더만 이동/삭제 — registry.json 영향 없음)
- `_DEPRECATED_260619_2_Daily_for_stock` 삭제 (완전 중복 확인됨)
- `260620_2_Daily_for_barobogi` → `_archive/` 로 이동 (registry에 등록 안 돼있어 자동화 안 건드림, git remote만 별도 확인)
- `GD/`, `MYBOX/` → 성격상 "프로젝트"가 아니라 개인 원본 백업이므로, `_personal/` 같은 별도 상위 폴더로 분리해 프로젝트 목록에서 시야적으로 제외

### C. n8n 안정화 확인 후 진행 (registry.json 경로 수정 + 워처 재시작 필요)
- `260623_1_study_all` → `Diary_for_Barobogi`로 통합 (이미 있는 통합안 v0.1 실행) 후 원본 폴더 삭제
- `260620_1_cowork_cli_automation` 아카이브 (Global_Define 승격 완료 재확인 후)
- `DEV_ITEM/`의 특허 관련 docx만 `11_특허아이디어/minor/`로 이관, `DEV_ITEM/`은 리뷰 기록 전용으로 정리
- 날짜 대신 성격이 드러나는 이름으로 리네이밍은 이번 라운드에서는 보류 (원본 CLAUDE.md 규칙: 신규 폴더부터 점진 적용, 기존은 그대로)

## 5. 실행 시 체크리스트 (만복 담당)

이동/삭제 전 항상:
1. 해당 폴더가 `.cowork-projects-registry.json`에 있는지 확인 → 있으면 경로도 같이 수정
2. 해당 폴더에 독립 `.git` + GitHub remote 있는지 확인 (있으면 이동보다 "그대로 두고 문서화"가 더 안전할 수 있음)
3. master_watch 재시작 필요 여부 판단
4. 이동 후 `NEXT_PROJECTS.md`, `AI_hub/shared/tasks.json`에 경로 참조 있으면 갱신
