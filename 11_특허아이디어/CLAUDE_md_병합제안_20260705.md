## 붙여넣기용 초안 ("사용자 기본 설정" 섹션 뒤에 삽입 추천)

```markdown
## 🔧 만복 실행 원칙 (2026-07-05 추가)

- **단순함 우선**: 요청 범위를 벗어나는 기능·추상화·설정가능성 추가 금지. 200줄로 짤 걸 50줄로 짤 수 있으면 다시 짤 것.
- **외과적 수정**: 관련 없는 코드/포맷은 건드리지 않음. 내 변경으로 생긴 미사용 import/변수만 정리하고, 원래 있던 죽은 코드는 삭제하지 말고 언급만 할 것.
- **파일 헤더 한글 주석**: 새 소스 파일 첫 줄에 역할을 한 줄 한글 주석으로 남길 것 (config 파일류는 제외). 다른 세션/AI가 맥락 파악하기 위함.
- **목표주도 실행**: 비트리비얼 작업은 "단계 → 검증 방법" 형태로 짧게 계획 후 진행.
- **완료 전 테스트**: 코드를 건드렸으면 "완료" 보고 전에 반드시 실행/테스트 확인. 테스트 셋업이 없으면 최소 빌드/실행 확인.
- **시맨틱 커밋**: 한 문장으로 설명 가능한 논리적 단위마다 커밋. 여러 성격이 섞이면 나눠서 커밋. (프로토타입/일회성 스크립트는 느슨하게 묶어도 됨)
- **에러는 읽고 판단, 추측 금지**: 에러/로그 원문을 확인한 뒤 원인을 판단할 것. "흔한 원인일 것 같다"고 패턴매칭으로 먼저 고치지 말 것.

## 📋 작업 시작 시 체크리스트 (REF 규칙의 짝)

비트리비얼 작업 시작 시, 종료 시 REF_continue.md 갱신처럼 시작 시에도:
- 체크리스트(`checklist.md`) — 할 일을 체크박스로, 진행하며 체크
- 컨텍스트 노트(`context-notes.md`) — 작업 중 내린 결정과 이유를 계속 追記

계획만 주어지고 바로 코딩하라고 하면, 먼저 "체크리스트/컨텍스트 노트부터 만들까요?" 확인할 것.
```
