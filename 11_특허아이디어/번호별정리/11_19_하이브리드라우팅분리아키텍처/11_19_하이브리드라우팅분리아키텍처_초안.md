# 11_19 — 하이브리드 라우팅 분리 아키텍처

**버전**: v0.1 (2026-07-18)
**상태**: 초안 등록 완료
**작성**: 안티

## 1. 핵심 아이디어

단일 텔레그램 봇 환경에서, 명시적 호출(``/vibe``)과 일반 대화 요청을 서로 다른 경로로 분기하여 개발망과 업무망을 분리한다. 개발 요청은 백그라운드 파이썬 서브프로세스로 즉시 격발하고, 일반 대화는 N-AI 레지스트리 기반 동적 라우팅으로 처리한다.

## 2. 기술 요약

- 명시적 호출과 일반 대화의 경로 분리
- 개발망과 업무망의 물리적/논리적 격리
- 봇 내부 프로세스 충돌 및 에이전트 혼동 방지
- 동적 라우팅 기반 업무 분산

## 3. 기대 효과

- 채널 혼선 최소화
- 개발 작업과 운영 업무의 안정적 분리
- 멀티 에이전트 환경에서 충돌 저감

## 4. 특허적 관점

단순한 채널 분리 메커니즘이 아니라, 호출 유형에 따라 서로 다른 처리 경로를 명시적으로 설계하여 시스템 안정성과 운영 효율을 동시에 확보하는 기술로 평가된다.
