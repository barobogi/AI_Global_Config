## 발명의 명칭
분산 AI 에이전트 간 상태 동기화 및 지식 검정·승격 시스템

## 기술 분야
복수의 이기종 AI 에이전트(상시 실행형과 세션 기반 실행형 혼재)가 사람의 수동 중개 없이 작업 상태와 지식을 상호 동기화하고, 동기화된 정보의 신뢰성·재사용성을 자동 평가하여 전역 공유 여부를 결정하는 분산 AI 협업 인프라.

## 발명의 배경
여러 AI 에이전트가 하나의 작업을 나누어 수행할 때, 종래에는 사람이 한 AI의 출력을 복사해 다른 AI에게 전달하는 순차 릴레이 방식에 의존했다. 문제점: (1) 사람 개입 없이는 병렬 협업 불가 (2) 이기종 에이전트 간 "생존 상태" 판별 표준 기준 없음 (3) 무분별 공유 시 출처·신뢰도·충돌 미검증으로 오류 정보 확산 위험.

## 핵심 구성요소

```
[Layer A] 개별 에이전트 상태 계층
  상시형: heartbeat + 이벤트 즉시 기록 / 세션형: 활성 플래그 + 세션 시작 시 브리핑 자동생성
       ↓
[Layer B] 공유 동기화 계층 (AI_hub)
  공유 저장소 직접 read/write / 비동기 메시지 채널 / 작업 큐(quorum 조합 지정)
       ↓
[Layer C] 지식 검정·승격 계층 (신규 핵심)
  독립 로그 수집·병합 → 매니페스트 생성 → 4기준 평가:
  (a) 출처확인 (b) 검증상태평가 (c) 충돌방지감지 (d) 재사용성판단
  → 전역승격 / 로컬유지 / 보류(사람판단요청) / 폐지
```

### 생존 판정 및 협업 모드 전환
상시형 활성+세션형 활성 → 병렬협업 / 상시형만 → 단독(누적) / 세션형만 → 단독(독립분석) / 둘다 비활성 → 대기

### 온보딩 브리핑
상시형 에이전트가 "마지막 세션형 활성 시점 이후 변경분"을 사전 요약 생성 → 세션형 에이전트가 전체 로그 순회 없이 즉시 최신 상태 파악

## 신규성 및 차별점

| 구분 | 기존 기술 | 본 발명 |
|---|---|---|
| AI 간 정보 전달 | 사람의 수동 복사·전달 | 공유 저장소를 통한 직접 동기화 |
| 생존 판정 | 단일 실행모델 가정 | 이기종(상시/세션) 별도 기준 + 협업모드 자동 전환 |
| 정보 신뢰성 관리 | 없음 | 출처·검증·충돌·재사용성 4기준 검정 후 선택적 승격 |
| 재개 시 컨텍스트 회복 | 전체 로그 재순회 | 사전 생성 브리핑으로 즉시 회복 |
| 확장성 | 대등 2자 가정 | Quorum 방식 3자 이상 확장 |

## 청구항 후보

- **독립항1(시스템)**: 이기종 실행방식 AI 에이전트 간 상태 동기화 시스템 — 생존판정모듈 / 수집모듈 / 검정승격엔진
- **종속항1(이기종 생존판정)**: 상시형=주기신호, 세션형=활성플래그 → 병렬/단독/대기 자동전환
- **종속항2(검정엔진 4기준)**: 출처확인/검증평가/모순탐지/활용가능성판단
- **종속항3(보류·인간판단요청)**: 보류 분류 시 별도 채널에 기록해 인간 최종결정 요청
- **종속항4(온보딩 브리핑 사전생성)**: 세션 기반 에이전트 재개 시 단일 브리핑으로 최신상태 획득 *(11_11 원류로 2026-07-03 흡수 확정)*
- **종속항5(Quorum 기반 확장)**: 작업단위별 필수/선택 참여 에이전트 조합 동적 지정 *(11_10 원류로 2026-07-03 흡수 확정)*

## 11_5(4계층 아키텍처)와의 비교 결론
완전히 같은 발명은 아니나 11_5가 특허3의 "선행 개념(원형)"처럼 읽힐 소지 있음. 특허3 명세서에 "11_5의 단일-출처 승격 방식과 달리, 본 발명은 복수 독립 AI 에이전트 간 충돌 조정과 출처 신뢰도 평가를 포함한다"는 비교 문단을 명시해 자기-선행기술 리스크를 사전 방어 + 진보성 근거로 활용 권고.

## 특허 가능성
중간~중상. 이기종 실행모델 간 생존판정+자동 협업모드 전환, 출처·충돌·재사용성 기반 지식 검정·승격 로직의 조합은 선행기술(공유메모리/RAG/멀티에이전트 오케스트레이션)에서 미발견. 순수 SW이므로 구체적 알고리즘(임계값, 평가함수)을 명세서에 구체화할수록 등록 가능성 상승.
