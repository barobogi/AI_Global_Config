# 11_17 — 다중 AI 에이전트 신규 제안 사전 중복탐지 + 첫실사용시나리오 강제 검증 파이프라인

**상태**: ⚠️ 1차 선행검색 완료(2026-07-12) — 신규성 MID~HIGH, 2차 선행검색(회사 AI) 대상으로 승격
**작성일**: 2026-07-12 | **브레인스토밍**: 코니 + 바로보기 (3AI 안티 합류 이후 신규 프로젝트 발굴 프로세스에서 도출) | **1차 선행검색**: 코니

## 핵심 아이디어

여러 AI 에이전트(만복/코니/안티, 서로 다른 도구·세션)가 협업하는 환경에서, 한 에이전트가 새 프로젝트/기능을 제안할 때:

1. 제안 내용을 기존의 **계층적 프로젝트 분류체계(뿌리체계, tasks.json 등 구조화된 레지스트리)**와 자동/반자동 대조하여 중복 여부를 구현 전에 판정.
2. 제안서에 **"첫 실사용 시나리오"(누가, 언제, 어떤 실제 업무에 쓸 것인가)가 구체적으로 명시되지 않으면 게이트를 통과시키지 않음** — 실사용 계획이 막연하면 반려하고 재작성을 요구.
3. 검증을 통과한 제안만 검토 담당 에이전트(중간 검토자)가 1차 보강(부족한 부분 채움)한 뒤 최종 결정 에이전트에게 전달, 최종 결정자가 채택 시 레지스트리에 반영 + 실행 지시서를 발급.
4. 사람(사용자)의 리뷰 완료 여부를 제안서 메타데이터(배지)로 명시해, 사람 리뷰 미완료 건은 다음 단계로 전달 자체가 차단됨.

즉, "구현 후 사용 여부를 감사하는" 사후 검증이 아니라, **구현 전 단계에서 중복·실사용성을 게이트하는 사전 검증 파이프라인**이 핵심.

## 발굴 배경 (실사례 기반, 2026-07-12)

3AI 체계(만복/코니/안티)에서 안티가 외부 콘텐츠(유튜브 영상) 분석을 통해 파생한 신규 프로젝트 제안 3건 중 2건이 기존 항목(진행 중인 보안패치 파이프라인, 프로세스 워치독)과 실질적으로 중복임이 사전검토 단계에서 발견되어 구현 전에 반려됨. 이전에는 구현 완료 후에야 "실사용 사례가 없다"는 사실이 감사에서 드러나는 패턴이 반복됐음(동일 세션에서 3개 프로젝트 모두 구현은 완료됐으나 실사용 0건인 사례 확인). 이 경험을 바탕으로 검증 시점을 기획 단계로 앞당기는 구조를 설계함.

## 1차 선행검색 결과

**검색 방법**: 구글 특허 + 학술자료, 영문 키워드
("AI agent project proposal duplicate detection taxonomy validation patent", multi-agent AI proposal validation, autonomous agent action validation patent 등)

**주요 발견**:
1. **US20260017525A1** "Validating autonomous artificial intelligence (ai) agents using generative ai" — 에이전트의 행동/제안을 사전정의된 경계·규제기준과 대조해 검증하는 시스템. 다만 이건 "에이전트 행동이 정책을 준수하는가"를 검증하는 것으로, 본 아이디어의 "제안이 기존 계층적 레지스트리와 중복되는가 + 첫 실사용 시나리오가 구체적인가"를 판정하는 것과는 검증 대상과 기준이 다름.
2. 일반적인 "Duplicate Detection Classifier"(중복 판별 분류기) 관련 특허/기법은 다수 존재하나, 대부분 텍스트/데이터 중복 탐지이지 "계층적 프로젝트 분류체계 대비 신규 제안의 중복 여부 + 실사용성 게이트"라는 조합은 확인되지 않음.
3. 멀티 에이전트 오케스트레이션 관련 특허들(예: US20250259044A1 등)은 에이전트 간 작업 분배·리소스 할당에 초점이 있고, "제안 단계의 중복탐지+실사용성 강제"라는 게이트 메커니즘은 별도로 다루지 않음.

**판정**: 신규성 **MID~HIGH**. 개별 요소(중복탐지, 에이전트 행동검증, 계층적 분류체계)는 각각 선행기술이 존재하지만, "다중 이종 AI 에이전트 협업 환경에서 신규 제안을 계층적 레지스트리 중복탐지 + 첫 실사용 시나리오 명시 강제라는 두 조건으로 구현 전 게이트하고, 사람 리뷰 완료 배지로 다음 단계 전달을 잠그는" 구체적 조합은 직접적으로 겹치는 선행기술을 찾지 못함.
