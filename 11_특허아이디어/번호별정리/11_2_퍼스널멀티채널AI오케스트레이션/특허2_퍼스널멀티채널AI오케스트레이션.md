## 발명 개요

복수의 이기종 AI 에이전트 채널을 단일 컨텍스트와 페르소나로 통합 운영하여, 채널별 역할을 분리하면서도 사용자 경험의 연속성을 보장하는 퍼스널 AI 오케스트레이션 시스템.

## 현재 구현 (바로보기+만복이 시스템)

```
[채널1] 텔레그램 봇(두복이) — 주식 신호 알림, 빠른 조회, 외출 중 모니터링
[채널2] 모바일 Claude(세복이) — 이동 중 원격 작업 지시
[채널3] 데스크탑 만복이(CLI) — 깊은 작업, 코드 작성, 파일 관리
       ↕ [공유 컨텍스트 레이어] Firebase 실시간 동기화 / 파일시스템(코니_sync.md, REF) / master_watch.py 자동 push
```

## 확장 구성 (예정)
채널1 텔레그램(알림) / 채널2 모바일(원격지시) / 채널3 데스크탑(깊은작업) / 채널4 n8n(자동화) / 채널5 이메일(백업알림) / 채널6 호두AI 인형(음성 인터페이스) / 채널7 스마트홈(IoT)
→ 채널 무관 동일 페르소나 유지, 컨텍스트 자동 인계, 역할 우선순위 자동 라우팅

## 핵심 차별점

| 구분 | 기존 멀티채널 봇 | 본 발명 |
|---|---|---|
| 페르소나 | 채널마다 별개 | 단일 페르소나 전 채널 유지 |
| 컨텍스트 | 채널 내 고립 | 채널 간 자동 인계 |
| 역할 | 동일 기능 반복 | 채널별 명시적 역할 분리 |
| 라우팅 | 사용자가 채널 선택 | AI가 상황에 따라 최적 채널 자동 선택 |

## 검토자 대응 논리 ("원격데스크탑이랑 뭐가 달라요?")

| | 원격 데스크탑 | 본 발명 |
|--|--|--|
| 공유 대상 | 화면(UI) | 두뇌(Context) |

**비유**: 원격 데스크탑은 "친구 집 컴퓨터 화면을 내가 조작" → 친구 집 전기 나가면 끝. 본 발명은 "같은 메모장을 공유"해서 한쪽이 꺼져도 다른 AI가 동일 기억으로 이어받음.

**기술적 핵심**: 컨텍스트(기억)를 실행 주체와 구조적으로 분리해 외부 독립 저장소에 유지 → 복수 AI 인스턴스가 동일 컨텍스트를 공유·인계 (화면공유로는 원천적으로 재현 불가능)

**권한 검증형 선택위임**: 데스크탑 CLI 접속=모든 권한 / 텔레그램 봇=알림·조회만 / 모바일 앱=제한 작업만 → 최소 컨텍스트 라우팅 + 결과 검증 병합 구조 필요
